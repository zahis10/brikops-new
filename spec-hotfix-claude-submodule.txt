# HOTFIX — Remove orphan .claude/worktrees submodule (unblocks Cloudflare Pages)

## What & Why
Cloudflare Pages builds for `app.brikops.com` have been FAILING silently on
every commit since `0849756 chore: app icons (iOS + Android) + signing config`.
Root cause confirmed from Cloudflare build log:

```
fatal: No url found for submodule path '.claude/worktrees/gifted-meitner'
       in .gitmodules
Failed: error occurred while updating repository submodules
```

The tree `.claude/worktrees/gifted-meitner` is committed as a **gitlink**
(mode `160000`, pointing to commit `bb830cc`) but there is **no**
`.gitmodules` file at the repo root. This is an orphan worktree reference
left behind by a Claude Code `isolation: "worktree"` agent run. It needs
to be removed from the git index AND blocked from coming back.

Zahi's iPhone ALSO shows the wrong splash because this commit-family has
never actually been served to any device via Cloudflare Pages. Unblocking
Cloudflare unblocks the whole pipeline.

Verification before fix — run and paste output:

```bash
git ls-tree HEAD .claude/worktrees/ | head -5
# Expected: one line starting with "160000 commit bb830cc..."

ls -la .gitmodules 2>&1
# Expected: "No such file or directory"
```

## Done looks like
- `git ls-tree HEAD -- .claude/` prints nothing.
- `.gitignore` contains `.claude/` so the worktree directory cannot be
  re-committed by accident.
- `git status --porcelain` is clean after the commit.
- After push to origin main, Cloudflare Pages dashboard shows a new build
  attempt that gets PAST the submodule step (may or may not succeed after
  that — we just need submodule step to stop failing).
- Capgo GitHub Action on same push uploads a new bundle to production
  channel (run +1 from current).
- No other file in the repo changes.

## Out of scope
- No changes to any source file under `frontend/` or `backend/`.
- No changes to `.github/workflows/`, `deploy.sh`, `ship.sh`,
  `capacitor.config.json`, native Xcode/Android projects.
- No bump to `versionCode` / `versionName`.
- No new npm/yarn/pip dependencies.
- Do NOT delete or modify any other committed file in `.claude/` if any
  exists. We ONLY touch `.claude/worktrees/` and `.gitignore`.

## Tasks

### Task 1 — Remove the orphan gitlink from the index

```bash
# Confirm the offending entry exists (should print one line, mode 160000)
git ls-tree HEAD .claude/worktrees/

# Remove it from the index (does NOT delete the local folder)
git rm --cached -r .claude/worktrees

# Double-check it's gone
git ls-files | grep '^\.claude/' || echo "clean — no .claude entries in index"
```

### Task 2 — Add `.claude/` to `.gitignore`

Append to the end of `.gitignore` (create the file if missing, but it
already exists at repo root — do NOT rewrite existing lines):

```
# Claude Code local state — never commit worktrees or agent scratch
.claude/
```

Verify:
```bash
tail -5 .gitignore
git check-ignore -v .claude/worktrees/gifted-meitner
# Expected: prints the matching .gitignore line
```

### Task 3 — Single commit, push, and create review.txt
