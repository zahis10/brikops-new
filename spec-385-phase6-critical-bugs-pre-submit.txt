## Wave 1a — Bug #4: iOS auto-zoom on input focus

### Root cause (confirmed)
`frontend/src/index.css` line 98 has a TYPO:
```
font--size: 16px !important;
```
It's `font--size` (double dash) instead of `font-size`. The rule is silently invalid,
so inputs fall back to default ~14px → iOS auto-zooms on focus.

### Fix — 1 line
File: `frontend/src/index.css` line 98
Change:
```
font--size: 16px !important;
```
to:
```
font-size: 16px !important;
```

Verify via:
```
grep -n "font-size\|font--size" frontend/src/index.css
```
Expected: only valid `font-size` declarations, no `font--size`.

### Viewport meta — verify, DO NOT change
File: `frontend/public/index.html` line 5
Current:
```
<meta name="viewport" content="width=device-width, initial-scale=1, viewport-fit=cover" />
```
This is correct. DO NOT add `user-scalable=no` or `maximum-scale=1`.

### VERIFY
1. Capacitor rebuild → install on iPhone → open any form with text input
2. Tap into input → page must NOT zoom in
3. Type → zoom does not change
4. Tap outside → zoom stays at 1.0


## Wave 1b — Bug #7: Offline state

### Root cause (confirmed)
No `navigator.onLine` or offline hook exists anywhere in codebase.
`/frontend/src/App.js` line 103 has `networkError` state in ProtectedRoute, but data-level
screens have no offline awareness → they just show red error "שגיאה בטעינת סיכום ליקויים"
then empty state "לא נמצאו נתונים לבניין זה".

### Fix

#### 1. Create hook
New file: `frontend/src/hooks/useOnlineStatus.js`
```javascript
import { useEffect, useState } from 'react';

export function useOnlineStatus() {
  const [online, setOnline] = useState(typeof navigator !== 'undefined' ? navigator.onLine : true);
  useEffect(() => {
    const on = () => setOnline(true);
    const off = () => setOnline(false);
    window.addEventListener('online', on);
    window.addEventListener('offline', off);
    return () => {
      window.removeEventListener('online', on);
      window.removeEventListener('offline', off);
    };
  }, []);
  return online;
}
```

#### 2. Create OfflineState component
New file: `frontend/src/components/OfflineState.js`
Simple centered card: WifiOff icon + "אין חיבור לאינטרנט" + hint + Retry button.
Use existing design tokens (amber primary, Heebo font, RTL).

#### 3. Wire into 4 priority screens
In each screen, before the existing error UI:
```jsx
const online = useOnlineStatus();
if (!online) return <OfflineState onRetry={refetch} />;
if (error) { ... existing error UI stays ... }
```

Priority screens:
- `frontend/src/pages/BuildingDefectsPage.js` (the one with "שגיאה בטעינת סיכום ליקויים")
- `frontend/src/pages/ProjectDashboardPage.js`
- `frontend/src/pages/ProjectsListPage.js` (or equivalent)
- `frontend/src/pages/ProjectControlPage.js`

grep to confirm file names:
```
grep -rn "שגיאה בטעינת סיכום" frontend/src/ --include="*.js"
grep -l "ProjectDashboard\|ProjectsList" frontend/src/pages/
```

DO NOT wire into every screen in the app — just the 4 above. Build 4 expands coverage.

3. Open a project → Dashboard → shows "אין חיבור לאינטרנט" (NOT red error)

WAVE 2 — Selectors (bugs #1, #2, #3)
========================================================================

### Why wait
Wave 1 changes input font-size 14 → 16px globally. This may shift form layouts and
change how the bottom-sheet selectors render. Must see the impact first.

### Root cause (confirmed)
There are TWO separate SelectField implementations:
- `frontend/src/components/NewDefectModal.js` lines 87-133 — SelectField using Radix Dialog as Sheet
- `frontend/src/pages/ProjectControlPage.js` lines 144-159 — separate SelectField with OptionsOverlay

Both use `position: fixed; inset-x-0 bottom-0` bottom-sheet pattern, which on iOS
Capacitor gets clipped by keyboard / safe area.

### Fix — TWO options, pick one

#### Option A (preferred) — Refactor to one shared component
1. Create `frontend/src/components/BottomSheetSelect.js` as the single source of truth
2. Fix the bottom-sheet positioning once:
   - Replace `100vh` with `100dvh` (dynamic viewport height) anywhere it appears
   - Add `max-height: calc(100dvh - 120px); overflow-y: auto;` to the sheet content
   - Use `visualViewport.height` to recompute height when keyboard opens
   - Respect safe-area: `padding-bottom: env(safe-area-inset-bottom);`
3. Replace both SelectField implementations with imports of the new component

#### Option B (fast path if A is too risky) — Fix both separately, same fix applied twice
Apply the same positioning fixes to both files independently.
Leaves tech debt (duplicated component) but lower risk.

**Replit — pick Option A unless you hit a blocker. If Option B, flag it in the report.**
