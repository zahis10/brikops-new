BUG B — UnitDetailPage tasks list doesn't refresh on return / focus
================================================================================
File: frontend/src/pages/UnitDetailPage.js

ROOT CAUSE
----------
Batch 5 #2 added refresh-on-focus + location.key listener to ProjectTasksPage
but missed UnitDetailPage. Same staleness pattern, same fix needed.

When user opens a task → closes it → goes back, UnitDetailPage retains its
state (React Router doesn't unmount on history-back if the route matches).
The list (`tasks`) and the unit data with KPI counts (`unitData.kpi`) are
both stale because `loadTasks` and `loadUnit` only run on mount + filter
changes.

FIX — Mirror the ProjectTasksPage Batch 5 #2 pattern
----------------------------------------------------

STEP 1: Add useLocation import at top of file:
   Find:  import { useParams, useNavigate, useSearchParams } from 'react-router-dom';
   Replace with: import { useParams, useNavigate, useSearchParams, useLocation } from 'react-router-dom';

   Also add useRef to the React imports at line 1:
   Find:  import React, { useState, useEffect, useCallback } from 'react';
   Replace with: import React, { useState, useEffect, useCallback, useRef } from 'react';

STEP 2: Inside the UnitDetailPage component, AFTER the existing
   `useEffect(() => { loadTasks(); }, [loadTasks]);` line (around line 113),
   ADD:

       // Refresh-on-return — refetch unit + tasks when user navigates back
       // to this page (location.key changes) or when window regains focus.
       // Mirrors the pattern in ProjectTasksPage. 500ms debounce to avoid
       // double-firing when both events fire at once.
       const location = useLocation();
       const latestLoadUnitRef = useRef(loadUnit);
       const latestLoadTasksRef = useRef(loadTasks);
       const lastRefreshAtRef = useRef(0);
       const didMountRef = useRef(false);
       useEffect(() => { latestLoadUnitRef.current = loadUnit; }, [loadUnit]);
       useEffect(() => { latestLoadTasksRef.current = loadTasks; }, [loadTasks]);

       useEffect(() => {
         const refresh = () => {
           const now = Date.now();
           if (now - lastRefreshAtRef.current < 500) return;
           lastRefreshAtRef.current = now;
           latestLoadUnitRef.current();
           latestLoadTasksRef.current();
         };
         if (didMountRef.current) {
           refresh();
         } else {
           didMountRef.current = true;
         }
         const onFocus = () => { refresh(); };
         const onVisibility = () => { if (document.visibilityState === 'visible') refresh(); };
         window.addEventListener('focus', onFocus);
         document.addEventListener('visibilitychange', onVisibility);
         return () => {
           window.removeEventListener('focus', onFocus);
           document.removeEventListener('visibilitychange', onVisibility);
         };
         // eslint-disable-next-line react-hooks/exhaustive-deps
       }, [location.key, unitId]);

- The KPI rendering or task list rendering JSX — they read from state which
