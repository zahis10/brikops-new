SCOPE
================================================================================

WHAT THIS BATCH DOES:

1. **Open SSO to all roles** — remove 4 hardcoded `role != 'project_manager'`
   restrictions that block contractors/management from using Google/Apple.
   Phone remains the canonical user identifier; SSO methods anchor to phone
   via OTP (already works in code — just needs the role gate removed).

2. **Capture terms-of-service consent** at signup.
   - Single mandatory checkbox: "אני מאשר/ת את תנאי השימוש".
   - Backend stores: terms_accepted_at (ISO timestamp), consent_ip.
   - Backfill all existing users with `terms_accepted_at = created_at`
     (test-only customers, no marketing consent ever, per Zahi).

3. **NO marketing consent** — explicitly excluded. BrikOps will never send
   promotional/marketing communications. Only operational/transactional
   notifications (defect assignments, reminders, status updates) which are
   exempt from Israeli spam law's prior-consent requirement.

WHAT THIS BATCH DOES NOT DO:
- Doesn't change WhatsApp template flow (still operational-only).
- Doesn't add marketing email infrastructure (no need).
- Doesn't change contractor invite flow — they still get WhatsApp invite,
  click link, register via the same unified page (just any auth method).
- Doesn't modify password requirements / 2FA / session management.
