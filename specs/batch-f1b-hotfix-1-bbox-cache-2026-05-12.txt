================================================================================
BATCH F.1b-hotfix-1 — Hit-test reads cached bbox + 44×44 min touch area
================================================================================
Date:           2026-05-12
Branch:         staging  (NOT main)
Severity:       P2 — F.1b shipped to prod 2026-05-12 but the drag-to-
                move feature does NOT activate on real devices. Zahi
                reports: tapping on an existing text label opens a
                NEW pendingText overlay instead of starting a drag.

                Root cause: `hitTestText` uses `ctx.measureText` to
                compute the pill bbox, but sets only `ctx.font` and
                NOT `ctx.direction='rtl' / textBaseline='middle' /
                textAlign='center'` like `redraw` does. On iOS WebKit
                (Capacitor's engine) with Hebrew text, `measureText`
                returns a different/narrower width depending on
                whether direction is set — bbox ends up too small.

                Additionally, even with correct bbox math, on mobile
                the pill is often smaller than a finger (~10px text
                in a ~25px pill). Per CLAUDE.md touch-target rule,
                minimum 44×44 display-px hit area is required.

Approach:       Two fixes, both surgical:
                  1. Cache the EXACT bbox computed by `redraw` directly
                     on the stroke object (`stroke._bbox`). Hit-test
                     reads from cache instead of re-measuring. Guarantees
                     hit-test matches what was visually rendered.
                  2. Expand cached bbox to a minimum of 44 DISPLAY-px
                     (= 44 / scale CANVAS-px) in either dimension if
                     smaller. Provides finger-friendly hit area.

Status:         AWAITING ZAHI APPROVAL — DO NOT DEPLOY

Files changed:  Frontend only:
                  - frontend/src/components/PhotoAnnotation.js
                    (~+10 LOC net: 1 cache write + ~10 LOC hit-test
                    body rewrite)
                Total: 1 file. No backend, no DB, no env vars, no
                new dependencies.

Baseline:       origin/staging HEAD has F.1b merged (commit after
                dee1879; verify with `git log`).

ANCHOR (find this block):
  const hitTestText = useCallback((pos) => {
    const canvas = canvasRef.current;
    if (!canvas) return -1;
    const ctx = canvas.getContext('2d');
    const scale = scaleRef.current || 1;

    for (let i = strokesRef.current.length - 1; i >= 0; i--) {
      const s = strokesRef.current[i];

      const padX = TEXT_PADDING_X / scale;
      const padY = TEXT_PADDING_Y / scale;

      ctx.save();
      ctx.font = `bold ${fontSize}px ${FONT_STACK}`;
      const metrics = ctx.measureText(s.text || '');
      ctx.restore();

      const rectWidth = metrics.width + padX * 2;
      const rectHeight = fontSize + padY * 2;
      const rectX = s.x - rectWidth / 2;
      const rectY = s.y - rectHeight / 2;

      if (
        pos.x >= rectX && pos.x <= rectX + rectWidth &&
        pos.y >= rectY && pos.y <= rectY + rectHeight
      ) {
        return i;
      }

REPLACE WITH:
  const hitTestText = useCallback((pos) => {
    // BATCH F.1b-hotfix-1 (2026-05-12) — read cached bbox from the
    // last redraw instead of re-measuring. Guarantees hit-test
    // matches the visible pill exactly. Falls back to a 44-display-
    // pixel minimum touch area around the stroke center for finger-
    // friendly tap targets (CLAUDE.md touch-target rule).
    const scale = scaleRef.current || 1;
    const minTouchCanvas = 44 / scale; // 44 display px → canvas px

    for (let i = strokesRef.current.length - 1; i >= 0; i--) {
      const s = strokesRef.current[i];
      if ((s.type || 'stroke') !== 'text') continue;

      let bx, by, bw, bh;
      if (s._bbox) {
        bx = s._bbox.x;
        by = s._bbox.y;
        bw = s._bbox.width;
        bh = s._bbox.height;

        // Expand to at least the minimum touch area if pill is small.
        if (bw < minTouchCanvas) {
          bx = s.x - minTouchCanvas / 2;
          bw = minTouchCanvas;
        }
        if (bh < minTouchCanvas) {
          by = s.y - minTouchCanvas / 2;
          bh = minTouchCanvas;
        }
      } else {
        // No cached bbox (shouldn't happen — redraw runs after every
        // commit). Defensive fallback: 44×44 around stroke center.
        bx = s.x - minTouchCanvas / 2;
        by = s.y - minTouchCanvas / 2;
        bw = minTouchCanvas;
        bh = minTouchCanvas;
      }

      if (
        pos.x >= bx && pos.x <= bx + bw &&
        pos.y >= by && pos.y <= by + bh
      ) {
        return i;
      }
    }
    return -1;
  }, []);

================================================================================
RISK
================================================================================
Low. Pure surgical fix.

- ⚠ `stroke._bbox` is a NEW field on text strokes. Backward compat:
  old text strokes saved in localStorage (if any) won't have it →
  defensive fallback path uses min 44×44 area. They'll be hit-
  testable from the first redraw onward (since redraw populates
  _bbox on every pass).
- ⚠ Mutating the stroke object during render is unusual but
  matches the established pattern (F.1b's moveStroke also mutates
  in place). React doesn't re-render based on stroke field changes
  because strokes are stored in state by reference; only state
  setter calls trigger re-render.
- ⚠ Min touch area COULD cause false positives if a label is in
  the corner near other UI. Mitigated: canvas is the only thing
  that gets touch events; toolbar buttons are siblings.
- ⚠ measureText removed from hit-test — perf win (one fewer
  per-touchstart canvas operation).

Bundle size: zero (no new imports).
Performance: hit-test is faster now (no measureText). Negligible
either way.
