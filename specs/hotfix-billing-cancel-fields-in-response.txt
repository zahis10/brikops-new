PROBLEM
-------
After a user cancels their subscription via POST /api/billing/org/{id}/cancel,
the backend correctly updates the DB (verified in MongoDB Atlas):
    auto_renew: false
    cancelled_at: <ISO date>
    cancelled_reason: "...some text..."
    expires_at: <ISO date>

But when the frontend re-fetches subscription state via GET /api/billing/org/{id},
the response's `subscription` object DOES NOT include cancelled_at,
cancelled_reason, or expires_at. So the frontend can't tell the sub was
cancelled and renders the WRONG state-machine card:
  - Expected: amber "המנוי בוטל" + "המשך מנוי" button
  - Actual: green "מנוי פעיל" without cancel button

User impact: nobody can see they cancelled, and nobody can reactivate via UI.

ROOT CAUSE
----------
backend/contractor_ops/billing.py:get_billing_for_org() — the function builds
the subscription dict with a hard-coded list of 16 fields (lines ~1090-1110).
The 3 new cancellation fields were added to the DB schema in the cancellation v1
batch but never added to this response builder.

Single file: backend/contractor_ops/billing.py
