PART 2 — docs/architecture/adr-XXX-*.md (one per major file)
================================================================================
Create one ADR per file identified in STEP 0.1 as >1500 lines. Target:
5-8 ADRs total. Each ADR follows this structure:

```
# ADR-NNN — <File or Module Name>

**Status:** Accepted (or Proposed / Superseded)
**Date:** YYYY-MM-DD
**File:** path/to/file.ext (LINE_COUNT lines as of date)

## Why this exists
[1-2 paragraphs: what this file does, why it's not split into smaller files]

## Architectural decisions
- Decision 1: [name]. Why: [reason]. Trade-off accepted: [trade-off].
- Decision 2: ...

## Conventions used here
- [convention 1, e.g. "All endpoints check get_current_user dependency"]
- [convention 2, e.g. "Status writes use string literals; reads use STATUS_BUCKET_EXPANSION"]

## Natural seams (where to split if/when refactor happens)
- Seam 1: [function/class/section name and line range]. Could become: [target file].
- Seam 2: ...

## When to refactor
- Trigger: [data point that would justify splitting, e.g. "If 60% of bugs in any quarter cluster in this file"]
- Pre-requisites: [what must be in place first, e.g. "Tests covering the 5 critical flows"]

## Recent changes
- YYYY-MM-DD (Batch X): [what changed and why]
- YYYY-MM-DD (Batch Y): ...
```

Length per ADR: 200-400 lines. Capture the WHY only — not exhaustive
documentation. The code is the source of truth for HOW; the ADR is the
source of truth for WHY.

Contents of the new section (target: <100 additional lines):

- Read docs/architecture/README.md before any spec implementation
  (especially when touching files that have an ADR)
- STEP 0 mandatory pre-flight grep audit (already standing rule)
- When touching a file in docs/refactor-backlog.md, propose folding the
  refactor into the current batch (Boy Scout Rule)
- Anti-patterns Zahi has flagged repeatedly:
    - WebkitOverflowScrolling: 'touch' → NEVER USE (iOS freeze)
    - ObjectId → NEVER USE (string UUIDs only)
    - now_il() → NEVER USE (UTC only)
    - console.log with PII → NEVER in production
    - modal={true} on Radix Dialog → can cause pointer-events: none on body
    - Photo annotation renders via createPortal → events bubble to parent
    - Native <select> in RTL + position:fixed drawer → escapes ~600px on
      Desktop browsers (mobile WebView is fine; use Radix Select for new code)
    - window.confirm() → ugly in mobile WebView; use Radix Dialog modal
      instead (reference: TaskDetailPage reopen confirm modal)
    - Local duplicates of constants → silently shadow imports. Always
      grep `^<CONST_NAME> = ` before assuming an update cascades.
- Where to find existing helpers (BEFORE building new ones):
    - check_org_billing_role (billing_router.py)
    - send_invoice_email (services/)
    - getProjectBackPath (utils/navigation.js)
    - STATUS_BUCKET_EXPANSION (tasks_router.py:40)
    - TERMINAL_TASK_STATUSES (constants.py:1)
    - tStatus / tCategory / tPriority (i18n module)
- Hebrew commit message format Zahi prefers:
    `fix(<scope>): <one-line summary in present tense, lowercase>`
- The standing rule: agent NEVER runs `./deploy.sh`, `./ship.sh`,
  `git commit`, `git push`. Stop at review.txt + .local/.commit_message
  + present_asset.

If the existing CLAUDE.md ends abruptly (no closing section), add a
