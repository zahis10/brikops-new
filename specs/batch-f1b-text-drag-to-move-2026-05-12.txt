                  - F.1c — tap-to-edit (deferred, ~50 LOC).

Approach:       Frontend-only. New "draggingTextRef" state + new
                hitTestText helper. The text-mode branch of
                startStroke gets a NEW pre-step: hit-test against
                existing text strokes first. If hit → start drag
                interaction. If miss → fall through to existing
                pendingText behavior (F).

                Movement threshold of 10px (per CLAUDE.md drag rule)
                differentiates tap from drag. Below threshold → no
                mutation (tap-only — reserved for F.1c edit).
                Above threshold → commit to drag, mutate stroke
                position on every move event.

                Drag mutates `strokesRef.current[i].x/y` directly +
                calls redraw. State sync to `setStrokes` happens
                ONCE at endStroke — avoids per-frame re-render (Mode
                B lesson from P1.1 pinch saga).

LOCATION 3 — Add hitTestText helper
================================================================================
File: frontend/src/components/PhotoAnnotation.js
Insert: BEFORE the `getPos` callback (around L226 on current HEAD).

ANCHOR (find this line):
  const getPos = useCallback((clientX, clientY) => {

ADD BEFORE it:
  // BATCH F.1b (2026-05-12) — hit-test against existing text labels.
  // Given a canvas-space position, return the index of the topmost
  // text stroke whose pill bbox contains the position, or -1 if none.
  // Iterates strokesRef in REVERSE order — last drawn = topmost = first to hit.
  // Math mirrors redraw's text-branch exactly (same scale, font, padding).
  const hitTestText = useCallback((pos) => {
    const canvas = canvasRef.current;
    if (!canvas) return -1;
    const ctx = canvas.getContext('2d');
    const scale = scaleRef.current || 1;

    for (let i = strokesRef.current.length - 1; i >= 0; i--) {
      const s = strokesRef.current[i];
      if ((s.type || 'stroke') !== 'text') continue;

      const sizeMultiplier = SIZE_MULTIPLIERS[s.size || 'medium'] || 1.0;
      const fontSize = (TEXT_FONT_SIZE * sizeMultiplier) / scale;
      const padX = TEXT_PADDING_X / scale;
      const padY = TEXT_PADDING_Y / scale;

      ctx.save();
      ctx.font = `bold ${fontSize}px ${FONT_STACK}`;
      const metrics = ctx.measureText(s.text || '');
      ctx.restore();

      const rectWidth = metrics.width + padX * 2;
      const rectHeight = fontSize + padY * 2;
      const rectX = s.x - rectWidth / 2;
      const rectY = s.y - rectHeight / 2;

      if (
        pos.x >= rectX && pos.x <= rectX + rectWidth &&
        pos.y >= rectY && pos.y <= rectY + rectHeight
      ) {
        return i;
      }
    }
    return -1;
  }, []);

Notes:
- `useCallback([])` — relies on refs only (strokesRef, scaleRef,
  canvasRef). Safe to memoize.
- `ctx.save/restore` around font set — same defense as redraw,
  prevents font leaking into subsequent canvas operations.
- Returns -1 (not null) for "no hit" so caller can check `>= 0`.

REPLACE WITH:
  const moveStroke = (pos) => {
    // BATCH F.1b — text drag branch (precedes the freehand path).
    if (draggingTextRef.current) {
      const drag = draggingTextRef.current;
      const dx = pos.x - drag.startTapX;
      const dy = pos.y - drag.startTapY;

      if (!drag.hasMovedPastThreshold) {
        // Squared distance check — avoids Math.sqrt per move event.
        const threshSq = DRAG_THRESHOLD_PX * DRAG_THRESHOLD_PX;
        if (dx * dx + dy * dy < threshSq) return;
        drag.hasMovedPastThreshold = true;
      }

      // Mutate in place — avoids per-frame re-render (Mode B lesson
      // from P1.1 pinch saga). Final state sync happens in endStroke.
      const stroke = strokesRef.current[drag.strokeIndex];
      if (stroke && (stroke.type || 'stroke') === 'text') {
        stroke.x = drag.origStrokeX + dx;
        stroke.y = drag.origStrokeY + dy;
        redraw(strokesRef.current);
      }
      return;
    }

    if (!drawingRef.current || !currentStrokeRef.current) return;
    currentStrokeRef.current = {
      ...currentStrokeRef.current,
      points: [...currentStrokeRef.current.points, pos],
    };
    setStrokes(allStrokes => {
      redraw([...allStrokes, currentStrokeRef.current]);
      return allStrokes;
    });
  };

RISK
================================================================================
Medium-low. Touches hot paths (move/end handlers) but additive — new
branches placed BEFORE existing branches, with early returns.

- ⚠ Hit-test math must EXACTLY mirror redraw math, or labels won't
  be tappable. Verified by reusing same SIZE_MULTIPLIERS, fontSize
  formula, padX/padY scaling, and center-aligned rectX/rectY.
- ⚠ Mutating strokesRef directly without setStrokes during drag
  works because redraw reads from strokesRef. State diverges from
  ref mid-drag — sync happens at endStroke. If anything reads
  setStrokes mid-drag (Save? Undo?) → would see stale state.
  Verified: Save reads strokesRef.current (line 287); Undo modifies
  state. Drag should not be interrupted by either.
- ⚠ Threshold of 10px in CANVAS coords. On a small mobile screen
  with scale ~0.3, that's 10/0.3 ≈ 33 canvas pixels but only 10
  display pixels. Standard accessibility threshold. Tuned with
  CLAUDE.md guidance.
- ⚠ If user lifts finger BEFORE crossing threshold = no-op (good).
  But pure tap on label could feel "broken" if user expected
  something. F.1c will repurpose this for edit. Until then,
  silence is intentional. Document in smoke test C explanation.
- ⚠ Drag + Save during gesture: Save button is at bottom of toolbar.
  If user accidentally taps Save mid-drag (unlikely — buttons are
  outside canvas area), endStroke wouldn't fire properly. Mitigated
  by: touchmove preventDefault on canvas blocks tap-through, and
  user must release on canvas before Save reaches.
- ⚠ touchcancel handler also calls endStroke — good, cleans up
  draggingTextRef if iOS interrupts the gesture.
- ⚠ Undo over a dragged label removes the label entirely (not
  restore to pre-drag position). Standard behavior for this
  architecture, documented in test I.
- ⚠ Cursor flip from grab → grabbing during drag won't be live
  on desktop (acceptable v1; caveat in LOCATION 8).

Bundle size: zero (no new imports).
Performance: 1 hit-test per touchstart in text mode; 1 redraw per
touchmove during drag. O(n) over strokes. Negligible for typical
photo annotation (≤10 strokes).

Comparison to F.1 (the broken attempt):
- F.1 added drag + size + edit = 286 LOC in one batch. Too many
  state interactions; broke the bundle.
- F.1b is +90 LOC of focused additive logic. Hit-test is a pure
  function. Drag state is a single ref. Touch handler interception
  is minimal (one branch per handler). Smaller blast radius.
