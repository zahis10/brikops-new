PROBLEM
-------
After cancellation v1 deploy (commit 52b173f), OrgBillingPage.js shows NOTHING
where the new state card was supposed to appear, for any subscription with
auto_renew=false that wasn't explicitly cancelled.

This affects:
  - All comped subscriptions (e.g. demo org 715e2420-e4f1-4ebd-81ed-b690e1f7f84d
    has auto_renew=false because it's a free comped sub valid until 2031)
  - Any sub where auto_renew is false for non-cancellation reasons (manual
    admin change, legacy data, etc.)

Real impact: in current prod state where ALL paying customers are 0 and most
orgs are trial/comped, NO ONE sees the new UI. The whole cancellation v1
feature is invisible.

ROOT CAUSE
----------
frontend/src/pages/OrgBillingPage.js lines ~1410-1490 — the IIFE has 3 mutually
exclusive states but the conditions don't cover the "active+auto_renew=false+
not cancelled" case:

    State 1 (Active+auto_renew):
        subStatus === 'active' && sub?.auto_renew !== false
        && !sub?.cancelled_at && paidUntilDate > now
        ↑ FAILS for comped (auto_renew=false)

    State 2 (Cancelled-active):
        !!sub?.cancelled_at && expiresAtDate > now
        ↑ FAILS (cancelled_at is null)

    State 3 (Expired):
        paidUntilDate <= now
        ↑ FAILS (paid_until is in the future)

    All 3 fail → IIFE returns null → nothing renders.

FIX
---
Single file: frontend/src/pages/OrgBillingPage.js around line 1410-1490.

Replace the State 1 block with a broader "active sub" check that ALWAYS
shows a green card when the sub is active+paid_until-future, but only shows
the cancel button when auto_renew=true.

CURRENT CODE (around line 1422-1486 — the State 1 condition + render):

    const isActiveAutoRenew = subStatus === 'active'
                              && sub?.auto_renew !== false
                              && !sub?.cancelled_at
                              && paidUntilDate
                              && paidUntilDate > now;
    ...
    if (isActiveAutoRenew) {
        return (
            <div className="bg-emerald-50 ...">
                ...
                <span>חידוש אוטומטי פעיל</span>
                <p>המנוי יתחדש אוטומטית ב-{formatDate(subPaidUntil)}</p>
                <button onClick={...}>ביטול חידוש אוטומטי</button>
            </div>
        );
    }

REPLACE WITH:

    // Renamed and broadened — covers comped/auto_renew=false subs too.
    // Cancel button is conditionally rendered only when auto_renew is on.
    const isActive = subStatus === 'active'
                     && !sub?.cancelled_at
                     && paidUntilDate
                     && paidUntilDate > now;
    ...
    if (isActive) {
        const isAutoRenewing = sub?.auto_renew === true;
        return (
            <div className="bg-emerald-50 border border-emerald-200 rounded-lg p-4 space-y-3">
                <div className="flex items-center gap-2">
                    <CheckCircle className="w-5 h-5 text-emerald-600" />
                    <span className="text-sm font-medium text-slate-700">
                        {isAutoRenewing ? 'חידוש אוטומטי פעיל' : 'מנוי פעיל'}
                    </span>
                </div>
                <p className="text-xs text-slate-500">
                    {isAutoRenewing
                        ? `המנוי יתחדש אוטומטית ב-${formatDate(subPaidUntil)}`
                        : `בתוקף עד ${formatDate(subPaidUntil)} — לא יחויב אוטומטית`}
                </p>
                {isAutoRenewing && (
                    <button
                        onClick={() => { setCancelReason(''); setShowCancelModal(true); }}
                        className="text-xs text-rose-600 hover:text-rose-700 font-medium"
                    >
                        ביטול חידוש אוטומטי
                    </button>
                )}
            </div>
        );
    }

KEEP UNCHANGED — State 2 (Cancelled-active, amber card) and State 3 (Expired,
rose card). Both are still correct.
