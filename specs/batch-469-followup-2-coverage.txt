FILE: frontend/src/pages/BuildingDefectsPage.js

         status: [],

       };

   REPLACE WITH:

         status: [],
         category: [],
         floor: [],

       };

LOCATION 2.2 — Add filter logic IMMEDIATELY AFTER the existing
   `filters.category` / `filters.status` apply blocks (around line ~182-192):

   FIND the existing apply block — it returns false if a filter excludes
   the unit/task. The exact shape depends on whether this page filters
   units (with aggregate counts) or individual tasks. STEP 0.5 confirms.

   IF the page aggregates by unit with counts (status object on each unit):
       At the unit-level filter — we cannot filter by is_safety at the
       aggregate level because counts don't track safety. SKIP unit-level
       filter. Move to LOCATION 2.3 — server-side filter on task fetch.

   IF the page filters individual tasks (rawItems):
       ADD after `filters.status` block:
           if (filters.is_safety === true && task.is_safety !== true) return false;

LOCATION 2.3 — Wire `is_safety` into the task fetch params. Find where
   `taskService.list` (or equivalent) is called and the params object
   is built:

   FIND something like:
       const params = { project_id: projectId, building_id: buildingId, ... };
       if (filters.status.length > 0) params.status = filters.status[0];
       ...

   ADD a parallel block:
       if (filters.is_safety === true) params.is_safety = true;

   IMPORTANT: the backend `is_safety` filter (added by #469 in
   tasks_router.py:271) only filters when the param is sent. Sending
   `false` would over-filter. Send only when true.

LOCATION 2.4 — Add the filter UI in the drawer/sidebar. STEP 0.3
   identifies the sidebar JSX. The current filter sections look like:

       <section>
         <h3>סטטוס</h3>
         {/* checkboxes for status options */}
       </section>
       <section>
         <h3>תחום</h3>
         {/* checkboxes for category */}
       </section>
       ...

   ADD a new section IMMEDIATELY AFTER the תחום (category) section:

       {/* Batch #469 followup-2 — safety toggle */}
       <section className="border-t border-slate-100 pt-4 mt-4">
         <label className="flex items-center justify-between gap-2 cursor-pointer py-3">
           <span className="text-sm font-medium text-slate-700">
             🛡️ ליקויי בטיחות בלבד
           </span>
           <input
             type="checkbox"
             checked={filters.is_safety || false}
             onChange={(e) => setFilters(prev => ({ ...prev, is_safety: e.target.checked }))}

           />
         </label>
       </section>

   IMPORTANT:
   - Wrapper has py-3 → label height ≥44px (Capacitor compliance).
   - dir="rtl" — the parent likely already provides this; if not, the
     `flex justify-between` reverses correctly in RTL.
   - Same orange palette as the chip in ProjectControlPage (consistency).
