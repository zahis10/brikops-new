================================================================================
PROBLEM (Zahi prod 2026-05-07)
================================================================================

Concrete scenario:
  - Building 9 (no QC activity yet): all new template stages appear with
    cells synced correctly ✓
  - Building 8 (existing QC runs from before template change): floor-scope
    stages added to template DON'T appear in QC view → no items in DB →
    matrix cells stay empty even though column renders

Root cause located in qc_router.py:

  async def _get_template(db=None, *, project_id=None, run=None):
      if run:
          vid = run.get("template_version_id")
          if vid:
              # Returns the OLD template snapshot tied to this run
              return await db.qc_templates.find_one({"id": vid}, {"_id": 0})
      ...

When `get_or_create_floor_run` calls `_get_template(db, run=run, ...)` for
an EXISTING run, it pulls the template version that was active when the
run was created. New stages added to the project's qc_template AFTER the
run was created are invisible to that run.

`_backfill_missing_items` is correct logic — it iterates the (old) template
and adds any items it doesn't already have. But since the template it sees
is OLD, there's nothing new to add.

Result: admin template additions don't propagate to existing runs.
Matrix shows the new column (template-driven, live read at /matrix) but
no items exist → no QC activity possible → matrix cells stay empty
forever.

================================================================================
THE FIX — auto-upgrade run to current template on access
================================================================================

When loading an existing run via get_or_create_*_run:
  1. Load the CURRENT template by project_id (ignore run's pinned version)
  2. Compare current template id vs run.template_version_id
  3. If different → update the run doc:
     - run.template_version_id = current.id
     - run.template_upgraded_at = now
     - run.template_upgraded_by = user.id
     - Audit log entry
  4. Use the current template for _backfill_missing_items
  5. Existing items are preserved; only NEW items added

Why this is safe:
  - _backfill_missing_items is keyed on (stage_id, item_id). If an item
    already exists, it's not re-inserted. New items added cleanly.
  - Existing data UNCHANGED. Approved items stay approved. Rejected stay
    rejected. Audit chains intact.
  - If admin REMOVES a stage from template: items for that stage stay
    in the run (orphaned). Matrix doesn't render the column anyway
    (it reads visible stages from current template). Harmless.
  - If admin RENAMES a stage_id (very rare): old stage's items become
    orphaned, new stage's items get added. Cosmetic only.

PART 1 — NEW helper `_resolve_run_template_with_upgrade`
================================================================================

FILE: backend/contractor_ops/qc_router.py

Add a new helper near _backfill_missing_items / _get_template:

    async def _resolve_run_template_with_upgrade(
        db,
        run: dict,
        project_id: str,
        actor: dict,
    ) -> dict:
        """
        Load the CURRENT template for the project. If the run is pinned
        to an older template version, upgrade the run document to point
        to the current version and log an audit entry.

        Returns the current template dict.

        Idempotent: if the run is already on the current version, no
        DB write happens (just returns the template).

        This unblocks _backfill_missing_items to add stages/items that
        admin added to the template AFTER the run was created.
        """
        current = await _get_template(db, project_id=project_id)
        current_id = current.get("id")
        run_version_id = run.get("template_version_id")

        if run_version_id == current_id:
            # Already on current — no upgrade needed
            return current

        # Upgrade the run to current template
        now = _now()
        actor_id = actor.get("id") if actor else None
        actor_name = (actor or {}).get("name") or "system"
        await db.qc_runs.update_one(
            {"id": run["id"]},
            {
                "$set": {
                    "template_version_id": current_id,
                    "template_id": current.get("template_id") or current_id,
                    "template_upgraded_at": now,
                    "template_upgraded_by": actor_id,
                    "template_upgraded_from": run_version_id,
                },
            },
        )
        # Refresh local copy for caller
        run["template_version_id"] = current_id
        run["template_id"] = current.get("template_id") or current_id
        run["template_upgraded_at"] = now
        run["template_upgraded_by"] = actor_id

        # Audit (best-effort — don't block on failure)
        try:
            await _audit(
                db,
                project_id=project_id,
                actor_id=actor_id,
                actor_name=actor_name,
                action="qc_run_template_upgraded",
                entity_type="qc_run",
                entity_id=run["id"],
                payload={"from": run_version_id, "to": current_id},
            )
        except Exception as e:
            logger.warning(f"[#601] audit failed for run={run['id']}: {e}")

        logger.info(
            f"[#601] upgraded run={run['id']} "
            f"template {run_version_id} → {current_id}"
        )
        return current
