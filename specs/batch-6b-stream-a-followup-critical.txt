----- BUG A — DIGEST FLOOD BY (project, company) instead of (project, user) -----

Root cause discovered in `send_all_contractor_reminders` (reminder_service.py:458):

  for project in projects:
      open_tasks = await _db.tasks.find({"project_id": pid, ...company_id...})
      company_ids = list(set(t["company_id"] for t in open_tasks))
      for cid in company_ids:
          result = await send_contractor_reminder(pid, cid, ...)

Then `send_contractor_reminder(project_id, company_id)` queries:
  tasks = _db.tasks.find({"project_id": pid, "company_id": cid, ...open...})

If a contractor user X has tasks tagged with company_id=A AND company_id=B in
the same project (common scenario — contractor employed by 2 sub-contracting
companies on same project), the OUTER loop runs twice for that project:
  1. send_contractor_reminder(project, A) → finds X → sends "1 ליקוי" (only A's tasks)
  2. send_contractor_reminder(project, B) → finds X → sends "1 ליקוי" (only B's tasks)

Result: contractor X gets 2 separate messages instead of 1 saying "2 ליקויים".

The pinned test test_send_all_three_projects_three_messages (added in V2 spec)
specifically tested 3 PROJECTS with same user — passed. It did NOT test the
same project + multiple companies scenario, which is exactly Bug A.

    # Stream A v2 follow-up (Batch 6B) 2026-05-03 — eligibility helpers

    DORMANT_USER_DAYS = 45  # Per Zahi 2026-05-03 — buffer for vacations.

    def _is_user_dormant(user: dict) -> bool:
        """User is dormant if their LAST_LOGIN_AT is older than DORMANT_USER_DAYS.

        IMPORTANT V2 fix (Replit feedback 2026-05-03): users who NEVER logged in
        must NOT be classified as dormant if their account is recent — otherwise
        we create a deadlock: new contractor invited → never logged in →
        marked dormant → no WA reminder → never logs in → permanent dormancy.

        Logic:
        1. If last_login_at exists: dormant iff > DORMANT_USER_DAYS ago.
        2. If last_login_at missing AND created_at recent (≤ DORMANT_USER_DAYS ago):
           → NOT dormant (give new user a chance to receive their first reminder).
        3. If last_login_at missing AND created_at also old (or missing):
           → dormant (truly inactive, safe to skip).
        """
        last_login_iso = user.get('last_login_at')
        now = datetime.now(timezone.utc)

        if last_login_iso:
            try:
                last = datetime.fromisoformat(last_login_iso.replace('Z', '+00:00'))
                return (now - last).days > DORMANT_USER_DAYS
            except Exception:
                pass  # Malformed → fall through to created_at check

        # No (valid) last_login → fallback to created_at
        created_iso = user.get('created_at')
        if created_iso:
            try:
                created = datetime.fromisoformat(created_iso.replace('Z', '+00:00'))
                # Recent account → not dormant (new user, give them a chance)
                return (now - created).days > DORMANT_USER_DAYS
            except Exception:
                return True  # Malformed everything → dormant

        return True  # No login + no created_at → safely classify as dormant

    def _user_disabled_whatsapp(user: dict) -> bool:
        """Honor user's WA toggle in settings (auth_router.py:144).
        Default True (notifications enabled) if field missing — preserves
        existing behavior for users created before the toggle was added."""
        return user.get('whatsapp_notifications_enabled') is False

    def _accumulate_summary(summary: dict, result: dict):
        """Helper to fold a per-recipient result into the cron summary counter."""
        status = result.get("status", "")
        if status == "completed":
            for r in result.get("results", []):
                if r.get("status") == "sent":
                    summary["sent"] += r.get("messages_sent", 1)
                elif r.get("status") == "failed":
                    summary["failed"] += 1
                elif r.get("status") == "skipped":
                    summary["skipped"] += 1
        elif status == "skipped":
            summary["skipped"] += 1
        else:
            summary["failed"] += 1

        cutoff = (_now_dt() - timedelta(hours=COOLDOWN_HOURS)).isoformat()
        existing = await _db.reminder_log.find_one({
            "type": {"$in": ["contractor_reminder", "contractor_reminder_digest"]},
            "project_id": project_id,
            "company_id": company_id,
            "recipient_user_id": None,  # only fallback entries
            "sent_at": {"$gte": cutoff},
            "status": {"$ne": "failed"},
        }, {"_id": 0, "id": 1})
        return existing is not None

Notes on the new design:
- The PRIMARY path (per-user aggregation) handles 100% of current prod data.
- The FALLBACK path is future-proofing: if a defect is created without an
  assignee but with a company, the company gets a single digest message.
- If the company has no phone in the system, the orphan defects are silently
  skipped (logged, not sent). This is correct — no spam to nowhere.
- Both paths share the same template, same URL pattern, same Hebrew copy.
- The company-fallback skips dormant + WA-toggle checks because those
  apply to USERS, not generic company contacts.
