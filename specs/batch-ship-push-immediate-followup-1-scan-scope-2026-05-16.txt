LOCATION 1 — Add directory/extension skip to secret-scan
================================================================================
File: ship.sh

In the secret-scan loop (the `while IFS= read -r f` block), add a
SKIP check BEFORE the binary file check. The current logic skips
binaries; we need to also skip documentation paths.

CURRENT (approximate, may differ slightly):
  while IFS= read -r f; do
    [ ! -f "$f" ] && continue
    file -b "$f" 2>/dev/null | grep -qE "binary|ELF|image|video" && continue
    # ... pattern checks ...
  done <<< "$STAGED_FILES"

ADD this skip immediately after the `[ ! -f "$f" ] && continue` line:

  # Skip documentation paths (contain pattern descriptions, not real secrets)
  case "$f" in
    specs/*|docs/*|future-features/*|memory/*|security/*) continue ;;
    ROADMAP.md|CLAUDE.md|README.md) continue ;;
    *.md) continue ;;
  esac

This uses shell `case` which is POSIX-portable and fast. It skips:
  - Any file under: specs/, docs/, future-features/, memory/, security/
  - Three specific top-level docs: ROADMAP.md, CLAUDE.md, README.md
  - Any other .md file (assumed documentation)

Documentation files CAN'T contain runtime secrets that the app uses.
If they have actual secrets (e.g., a leaked credential pasted into
a notes doc), other tooling (gitleaks, GitHub secret scanning) catches
them. ship.sh's job is to prevent ACCIDENTAL commits of config/source
files with secrets.
