LOCATION 1 — NEW: BulkPlanUploadModal component
================================================================================
File: `frontend/src/components/BulkPlanUploadModal.jsx` (NEW)
Approach: extract the existing bulk modal from ProjectPlansPage
verbatim, then parameterize. Internal state stays local to the
component (no callback prop dance for file list — caller doesn't
need to know).

PROP INTERFACE:
  {
    // Open/close control
    open: boolean,
    onClose: () => void,

    // Discipline config
    allDisciplines: Array<{value: string, label: string}>,  // dropdown options
    defaultDiscipline: string,                                // initial value
    onDefaultDisciplineChange: (value: string) => void,       // bubble up if needed

    // Per-file fields config
    showFloorField: boolean,                                  // true for ProjectPlansPage, false for UnitPlansPage
    floorsList?: Array<{id: string, name: string}>,           // required if showFloorField

    // Plan type config — pass options inline (no shared constant
    // exists; PLAN_TYPES is NOT a defined constant per STEP 0.8).
    // Expected default value: [
    //   { value: 'standard', label: 'כללית' },
    //   { value: 'tenant_changes', label: 'שינויי דיירים' }
    // ]
    planTypes: Array<{value: string, label: string}>,

    // Upload function — the page passes its bound service call
    uploadFn: async (file: File, discipline: string, opts: {
      plan_type?: string, floor_id?: string
    }) => Promise<{id: string, ...}>,

    // Success callback (e.g., refetch list)
    onUploadComplete?: () => void,

    // Optional limits
    maxFiles?: number,            // default 20
    maxFileSizeMb?: number,       // default 50
    acceptedExtensions?: string,  // default '.pdf,.jpg,.jpeg,.png,.dwg,.dxf'
  }

COMPONENT STRUCTURE (~280 LOC):
  - Internal state:
      const [files, setFiles] = useState([]);  // [{file, discipline, plan_type, floor_id, status, error, id}]
      const [uploading, setUploading] = useState(false);
      const [results, setResults] = useState(null);
  - useEffect to RESET state when `open` flips false→true:
      useEffect(() => {
        if (!open) {
          setFiles([]);
          setUploading(false);
          setResults(null);
        }
      }, [open]);
  - Helper: `handleFileSelect(fileList)` — append to `files` with default discipline applied + status='pending'. Validates size/count.
  - Helper: `removeFile(id)` — splice.
  - Helper: `updateFileMeta(id, field, value)` — update one row's discipline/plan_type/floor_id.
  - Helper: `handleBulkUpload()` — concurrency=3 loop. For each file, set status='uploading' → call `uploadFn(file, discipline, opts)` → set status='done'/'error'. Final: setResults({success: N, error: M}).
  - Helper: `retryFile(id)` — re-upload a single file in error state.
  - JSX:
      - Header: title "העלאה מרובה" + close X
      - Default discipline dropdown (single)
      - File picker (drag-drop + button, `multiple`)
      - File list: per-row with name, size, discipline override, plan_type override, floor_id override (if `showFloorField`), status pill, remove button
      - Footer: "העלה N קבצים" button + "ביטול" button
      - Post-upload results view: "X הצליחו, Y נכשלו" + retry buttons per failed row
  - Tailwind styling: dark theme matching ProjectPlansPage existing
    modal (slate-800/slate-700, amber primary, red destructive,
    safe-area-bottom padding for iOS).
  - `dir="rtl"` on outermost div.

Note: the implementation is essentially a COPY of the existing
ProjectPlansPage bulk modal logic, with the `floor_id` field
conditionally rendered based on `showFloorField` prop, and the
upload call delegated to `uploadFn` (instead of hardcoded
`projectPlanService.upload`).
