WHY
---
1. Israeli consumer law (חוק הגנת הצרכן 14ג2) requires online cancellation for
   any recurring subscription, with confirmation in writing. Current state:
   no online cancellation. Customer must contact Zahi manually. Legal exposure.
2. Current OrgBillingPage shows "חידוש אוטומטי — בקרוב" placeholder UI which
   is both misleading (we DO auto-renew) and dead weight. Removed in this batch.
3. Two wins in one batch: legal compliance + dead UI removal.

CRITICAL PRINCIPLE — ZERO PAYPLUS API CALLS IN THIS BATCH
----------------------------------------------------------
Cancellation is purely local DB state changes. The PayPlus token stays saved.
The renewal cron skips charging if `auto_renew=false`. This protects the
fragile billing stack from any new PayPlus integration risk.

SCHEMA — 3 new fields on subscriptions
================================================================================
auto_renew already exists; we just add the 3 cancel-related fields.

    {
      // existing fields (auto_renew, status, paid_until, etc.)
      "cancelled_at":     None,    // ISO datetime or None
      "cancelled_reason": None,    // string (max 500 chars) or None
      "expires_at":       None,    // ISO datetime or None
                                   //   = snapshot of paid_until at cancel time
                                   //   (we keep this even though paid_until
                                   //    has the same value — explicit signal
                                   //    that "this sub is in cancelled-but-
                                   //    active state, expiring at this time")
    }

State derivation (all from existing+new fields, no new status value needed):

    STATE             | auto_renew | cancelled_at | paid_until vs now
    ------------------|------------|--------------|-------------------
    Active            | True       | None         | future
    Cancelled-active  | False      | <set>        | future
    Expired           | (any)      | (any)        | past

The paywall already blocks the "Expired" state via _resolve_access.

POST /api/billing/org/{org_id}/cancel
--------------------------------------
Request body (optional):
    { "reason": "free-text from textarea, max 500 chars" }

Authorization: same helper as /checkout.
    - Reuse the check_org_billing_role + _is_super_admin pattern from
      billing_router.py:~163.
    - billing_admin OR super_admin → allowed.
    - Anyone else → 403 "אין הרשאת ניהול חיוב".

Logic:
    1. Auth check (as above).
    2. Find active sub:
         sub = await db.subscriptions.find_one(
             {'org_id': org_id, 'status': 'active'}, {'_id': 0}
         )
       If none → 404 "אין מנוי פעיל לביטול".
    3. Idempotency: if sub.auto_renew is False → return 200 with
         { "already_cancelled": True,
           "cancelled_at": sub.cancelled_at,
           "access_until": sub.expires_at }
       (Idempotent operations are NOT errors.)
    4. Get paid_until = sub.get('paid_until').
       Edge case: if paid_until missing/null → 500 "שגיאה בנתוני המנוי,
       צרו קשר עם התמיכה" + log error. Do NOT cancel without expires_at.
    5. Atomic update:
         await db.subscriptions.update_one(
             {'id': sub['id']},
             {'$set': {
                 'auto_renew': False,
                 'cancelled_at': now_iso,
                 'cancelled_reason': (body.reason or '').strip()[:500] or None,
                 'expires_at': sub['paid_until'],
             }}
         )
    6. Audit event (try/except — log on failure, don't fail flow):
         await db.audit_events.insert_one({
             'id': str(uuid.uuid4()),
             'event_type': 'subscription_cancelled',
             'org_id': org_id,
             'user_id': user['id'],
             'created_at': now_iso,
             'details': {
                 'sub_id': sub['id'],
                 'plan_id': sub.get('plan_id'),
                 'expires_at': sub['paid_until'],
                 'reason': body.reason or None,
             },
         })
    7. Send confirmation email to user (try/except):
         await send_cancellation_email_user(org_id, sub, user, body.reason)
    8. Send notification email to Zahi (try/except):
         await send_cancellation_email_zahi(org_id, sub, user, body.reason)
    9. Return:
         { "cancelled_at": now_iso,
           "access_until": sub['paid_until'] }

POST /api/billing/org/{org_id}/reactivate
------------------------------------------
Request body: empty.

Authorization: same as cancel (billing_admin OR super_admin).

Logic:
    1. Auth check.
    2. Find sub:
         sub = await db.subscriptions.find_one({'org_id': org_id}, {'_id': 0})
       If none → 404 "לא נמצא מנוי".
    3. State checks:
       - If sub.cancelled_at is None → 400 "המנוי כבר פעיל".
       - Get expires_at = sub.expires_at (or fall back to sub.paid_until).
       - If expires_at and now > expires_at → 410 "המנוי כבר פג, נדרשת
         רכישה חדשה". (HTTP 410 Gone — frontend uses this to redirect
         to checkout instead of showing an error.)
    4. Atomic update:
         await db.subscriptions.update_one(
             {'id': sub['id']},
             {'$set': {'auto_renew': True},
              '$unset': {
                  'cancelled_at': '',
                  'cancelled_reason': '',
                  'expires_at': '',
              }}
         )
    5. Audit event: subscription_reactivated.
    6. Send confirmation email to user (try/except):
         await send_reactivation_email_user(org_id, sub, user)
    7. Return:
         { "reactivated_at": now_iso,
           "next_charge_date": sub.get('paid_until') }

    const [showCancelModal, setShowCancelModal] = useState(false);
    const [cancelReason, setCancelReason] = useState("");
    const [cancelLoading, setCancelLoading] = useState(false);

    const handleCancelSubscription = async () => {
      setCancelLoading(true);
      try {
        await axios.post(
          `${BACKEND_URL}/api/billing/org/${orgId}/cancel`,
          { reason: cancelReason },
          { headers: { Authorization: `Bearer ${token}` } }
        );
        toast.success("המנוי בוטל. אישור נשלח לאימייל שלך.");
        setShowCancelModal(false);
        setCancelReason("");
        await refetchSubscription();
      } catch (e) {
        toast.error(e.response?.data?.detail || "שגיאה בביטול המנוי");
      } finally {
        setCancelLoading(false);
      }
    };

    const handleReactivate = async () => {
      if (!window.confirm("להפעיל את המנוי מחדש?")) return;
      try {
        await axios.post(
          `${BACKEND_URL}/api/billing/org/${orgId}/reactivate`,
          {},
          { headers: { Authorization: `Bearer ${token}` } }
        );
        toast.success("המנוי הופעל מחדש");
        await refetchSubscription();
      } catch (e) {
        if (e.response?.status === 410) {
          toast.error("המנוי פג, נדרשת רכישה חדשה");
          navigate(`/billing/org/${orgId}/checkout`);
        } else {
          toast.error(e.response?.data?.detail || "שגיאה בהפעלת המנוי");
        }
      }
    };
