LOCATION 1.2 — qc_router.py:911 replace lex sort with natural sort
================================================================================
File: backend/contractor_ops/qc_router.py
Line: 911

BEFORE:
-    units.sort(key=lambda u: u.get("unit_no", u.get("name", "")))

AFTER:
+    units.sort(key=lambda u: _natural_sort_key(u.get("unit_no") or u.get("name") or ""))

Why `_natural_sort_key` works for unit_no:
- "1" → [(0, 1)] → sorts as number 1
- "2" → [(0, 2)]
- "10" → [(0, 10)]
- "A1" → [(1, "A"), (0, 1)] → groups A-prefix together, sorted by number within
- "ק-901" → [(1, "ק-"), (0, 901)] → Hebrew prefix group sorted by trailing number
- "" → [(1, "")] (empty string fallback)

Result order: 1, 2, 10, 20, A1, A2, A10, ק-901, ק-1001 (correct natural sort).

LOCATION 2.1 — UnitQCSelectionPage.js: add explicit sort to unit aggregate
================================================================================
File: frontend/src/pages/UnitQCSelectionPage.js
Line: ~140-148 (the `const units = Array.from(unitAggMap.values()).map(u => {...})`
       block — find by anchor "const allApproved = u._statuses.length > 0")

BEFORE:
   const units = Array.from(unitAggMap.values()).map(u => {
     const allApproved = u._statuses.length > 0 && u._statuses.every(s => s === 'approved');
     const anyInProgress = u._statuses.some(s => s === 'in_progress' || s === 'approved');
     const status = allApproved ? 'approved'
                  : anyInProgress ? 'in_progress'
                  : 'not_started';
     const { _statuses, ...rest } = u;
     return { ...rest, status };
   });

AFTER:
   const units = Array.from(unitAggMap.values()).map(u => {
     const allApproved = u._statuses.length > 0 && u._statuses.every(s => s === 'approved');
     const anyInProgress = u._statuses.some(s => s === 'in_progress' || s === 'approved');
     const status = allApproved ? 'approved'
                  : anyInProgress ? 'in_progress'
                  : 'not_started';
     const { _statuses, ...rest } = u;
     return { ...rest, status };
   }).sort((a, b) => {
     // Natural sort (numeric-aware) — keeps "10" after "2" not after "1".
     // Hebrew locale so "ק-901" sorts predictably with Hebrew prefixes.
     const aKey = String(a.unit_no || a.unit_name || '');
     const bKey = String(b.unit_no || b.unit_name || '');
     return aKey.localeCompare(bKey, 'he', { numeric: true });
   });

Why localeCompare with `numeric: true` works:
- "1".localeCompare("10", 'he', {numeric:true}) → -1 (1 before 10)
- "2".localeCompare("10", 'he', {numeric:true}) → -1 (2 before 10)
- "A1".localeCompare("A10", 'he', {numeric:true}) → -1 (A1 before A10)
- "ק-901".localeCompare("ק-1001", 'he', {numeric:true}) → -1

This is browser-native and handles edge cases (Unicode normalization, RTL chars).
