CONTEXT
-------
After batch 505 + 505-hotfix shipped, contractors invited via WhatsApp can
register on /onboarding?invite=X but ONLY via phone+OTP+password. Non-invite
users on /onboarding (regular registration) DO have Google + Apple SSO
buttons (renderPhoneStep, L1054-1090). Zahi's product requirement:

  > "אבל זה לא מספיק טוב כי onboarding לא יכול להתחבר בתור קבלן.
  >  אני רוצה גם כשיש invite לקבלן או לאיש צוות
  >  תהיה לו האפשרות להירשם ישירות עם גוגל או אפל"

Decisions confirmed by Zahi:
  1. Phone-match OTP MANDATORY after Google OAuth — keeps 505 security
     posture. Forwarding an invite link + holding a Google account is
     not enough; the SMS round-trip on invite.target_phone is required.
  2. Buttons on the FIRST screen only (renderInvitePhoneStep, before phone
     entry) — parallel choice with the existing phone+OTP path.

Backend already handles this:
  - /auth/social with invite_token: validates invite, attaches to session
    (onboarding_router.py:1830-1839)
  - /auth/social/verify-otp register branch: enforces phone-match against
    invite.target_phone (L2135) and consumes invite (L2169+)

Frontend is the only missing piece. The existing handleGoogleSignIn and
handleAppleSignIn callbacks already pass inviteToken to socialAuth
(L200, L302) — the buttons just need to be rendered.

USER FLOW (after this spec):
  1. Contractor receives WhatsApp with invite link
  2. Click → /onboarding?invite=X → renderInvitePhoneStep
  3. Sees "יש לך הזמנה לפרויקט" + phone input + "שלח קוד אימות"
     **PLUS** divider "או הרשמה עם" + Google + Apple buttons
     **PLUS** ToS checkbox above SSO buttons (gates SSO + phone+OTP)
  4. Path A — phone: existing flow (505-hotfix added ToS at the post-OTP
     step; this spec adds it pre-OTP for SSO use too)
  5. Path B — Google:
     a. Click Google → standard GIS popup → user authenticates
     b. socialAuth('google', credential, null, inviteToken) → backend
        validates invite → returns registration_required + session_token
     c. handleSocialAuthResult sets socialFlow='register' AND setStep('phone')
        (the new line — was missing this so the SSO UI didn't show in
         invite mode after Google completed)
     d. renderPhoneStep socialFlow='register' branch (L1143) renders:
        phone input → user enters invite's target phone → /auth/social/
        send-otp → user receives SMS → enters OTP → /auth/social/verify-otp
        with invite_token + termsAccepted=true
     e. Backend: phone-match check against invite.target_phone (L2135),
        creates user with Google identity, joins project per invite, returns
        token. User lands on project tasks page.
