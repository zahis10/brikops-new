                Issue 2: After saving a photo with drawings, tapping
                the edit (pencil) icon on the thumbnail loads the
                ORIGINAL raw photo instead of the annotated version.
                User sees a clean image and loses their work
                visually. Reported by Zahi 2026-05-12: "אחרי ששמרתי
                תמונה עם ציור למשל, אני לוחץ על עריכה, זה מוחק את
                הציור ומשאיר לי תמונה נקיה".

                Root cause for Issue 2: `NewDefectModal.js:1080`
                feeds `originalFile || file` to PhotoAnnotation
                on re-annotate. The `||` priority loads `originalFile`
                (raw) FIRST when it exists. Flipping the priority
                to `file || originalFile` loads the ANNOTATED JPEG
                first → user sees rasterized drawings → can add new
                strokes on top.

                NOTE: this is the **intermediate fix**. User sees
                old drawings as pixels (rasterized), can paint over
                them, but cannot edit/undo individual previous
                strokes. The full architectural fix (persist strokes
                as JSON server-side) remains in backlog as
                "Re-editable annotations". For 80% of cases,
                Option A solves the user-facing pain.

  const handleDiscard = useCallback(() => {
    if (strokesRef.current.length > 0) {

    }

REPLACE WITH:

  // confirm modal instead of window.confirm. The modal's "אישור"
  // button then calls confirmDiscard which does the actual close.
  const handleDiscard = useCallback(() => {
    if (strokesRef.current.length > 0) {
      setShowDiscardConfirm(true);
      return;

    }

    if (onDiscardRef.current) {
      onDiscardRef.current();
    } else {
      onSaveRef.current(null, false);
    }
  }, []);

- ⚠ Custom modal renders absolutely-positioned inside the
  PhotoAnnotation portal. `z-50` is relative to the parent's
  stacking context. PhotoAnnotation's portal root is at `zIndex:
  10001`, so the modal sits above the canvas naturally.

  already-stored image. No need to flip priority elsewhere.

  cannot edit/drag/undo them individually. This is the intermediate
  fix. Full architectural fix ("Re-editable annotations") remains
  in backlog. Document in commit message.
