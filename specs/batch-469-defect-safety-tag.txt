CONTEXT
================================================================================

Per Phase 1 concept doc (`safety-and-worklog-concept.md:778`):
"Defect 'בטיחות' tag + pill + megaphone + 'סגור' affordance | 2 ימים"

Today, defects (tasks collection) have no way to be marked as
safety-related. PMs who want to flag a defect as safety-relevant for
internal awareness, supervisor reporting, or future safety dashboard
aggregation, have no tool — they end up putting "בטיחות" in the title
or description, which doesn't surface in any structured way.

This batch adds a simple boolean tag (`is_safety`), exposes it in the
defect creation modal, displays a pill on defect cards + detail view,
and supports filtering. NO new router file (per CLAUDE.md pattern —
extension, not new domain).

OUT OF SCOPE for this batch (separate sub-batches later):
- Megaphone notification (alert when safety defect created) — fits
  with Stream C bell wiring or future notifications batch.
- Safety dashboard widget showing safety-tagged defects count — fits
  with #471 dashboard polish.
- "סגור" affordance (special close flow if needed for safety) —
  separate UX polish if needed.
- Auto-tag based on category keywords — not in concept; manual only.
