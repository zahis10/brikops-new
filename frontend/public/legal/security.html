אבטחה ופרטיות מעודכן

עדכון אחרון: אפריל 2026

ב-BrikOps אנו מתייחסים לאבטחת המידע שלכם ברצינות המרבית. עמוד זה מפרט את אמצעי האבטחה, ההצפנה והגנת הפרטיות שאנו מיישמים כדי לשמור על הנתונים שלכם.

תשתית ואחסון

☁️ שרתי ענן

AWS Frankfurt (eu-central-1) — אחסון בתוך האיחוד האירופי בהתאם לדרישות GDPR.

🗄️ מסד נתונים

MongoDB Atlas EU — מוצפן במנוחה (AES-256) ובתעבורה (TLS 1.2+).

📁 אחסון קבצים

Amazon S3 עם Versioning מופעל — כל קובץ נשמר עם גרסאות קודמות.

🌐 CDN ואבטחת רשת

Cloudflare — הגנה מפני DDoS, WAF, ו-DNS מאובטח.

הצפנה

  • בתעבורה: כל התקשורת מוצפנת באמצעות HTTPS/TLS 1.2 ומעלה. אין תקשורת לא מוצפנת.
  • במנוחה: נתוני מסד הנתונים מוצפנים ב-AES-256. קבצים ב-S3 מוצפנים בצד השרת (SSE-S3).
  • סיסמאות: אין — אנו משתמשים באימות OTP בלבד (קוד חד-פעמי בSMS). אין סיסמאות שנשמרות במערכת.

אימות והרשאות

  • OTP (קוד חד-פעמי): כניסה באמצעות קוד SMS — ללא סיסמאות.
  • Google / Apple Sign-In: אימות חיצוני מאובטח כערוץ כניסה נוסף.
  • JWT Tokens: טוקנים חתומים עם תוקף מוגבל — מתחדשים אוטומטית.
  • Session Revocation: ניתן לנתק את כל ההתחברויות הפעילות מהגדרות החשבון.
  • RBAC: בקרת גישה מבוססת תפקידים — owner, project_manager, viewer, contractor. כל משתמש רואה רק את המידע הרלוונטי אליו.

Security Headers

  • HSTS: Strict-Transport-Security מופעל — הדפדפן תמיד יתחבר ב-HTTPS.
  • CSP: Content-Security-Policy מוגדר — חסימת הזרקת קוד זדוני.
  • X-Frame-Options: DENY — מניעת הטמעה ב-iframe (הגנה מפני Clickjacking).
  • X-Content-Type-Options: nosniff — מניעת MIME sniffing.

Rate Limiting והגנת API

  • Rate limiting גלובלי: הגבלת בקשות ברמת IP למניעת התקפות brute-force.
  • Rate limiting per-endpoint: הגבלות ספציפיות על endpoints רגישים (OTP, כניסה, יצירת חשבון).
  • HMAC Validation: כל webhook נכנס (WhatsApp, PayPlus) מאומת באמצעות חתימת HMAC.

גיבויים והתאוששות

  • גיבוי יומי: גיבוי אוטומטי מלא של מסד הנתונים מדי יום.
  • שמירה: 30 יום של גיבויים זמינים.
  • Point-in-Time Recovery: ניתן לשחזר את המערכת לכל נקודת זמן ב-30 הימים האחרונים.
  • S3 Versioning: קבצים שנמחקו ניתנים לשחזור מגרסאות קודמות.
  • Cross-region: גיבוי S3 משוכפל לאזור נוסף באירופה.

בדיקות אבטחה

  • בדיקת חדירה: בוצעה באפריל 2026. כל הממצאים תוקנו.
  • סקירת קוד: סקירה שוטפת של קוד חדש לפני העלאה לפרודקשן.
  • תלויות: סריקה תקופתית של ספריות צד שלישי לפרצות ידועות.

פרטיות ומחיקת נתונים

  • GDPR: המערכת פועלת בהתאם לעקרונות GDPR — איסוף מידע מינימלי, שקיפות, זכות מחיקה.
  • מחיקת חשבון: self-service מתוך האפליקציה. תקופת חסד של 7 ימים לביטול.
  • שמירת נתונים: 30 יום למידע אישי לאחר מחיקה. 7 שנים לרשומות ביקורת, חיוב ופרוטוקולי בנייה בהתאם לחוק.
  • אין מכירת מידע: איננו מוכרים, משתפים או מעבירים מידע אישי לצדדים שלישיים לצורכי פרסום.
