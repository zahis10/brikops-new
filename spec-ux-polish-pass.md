# #UX-002 — מעבר ליטוש: אייקונים, עקביות, ו-Empty States

## What & Why
ביקורת UX מתאריך 8.4.2026 מצאה ממצאים קטנים שביחד משפיעים על תחושת המקצועיות. עיקר הבעיה: שימוש באימוג'ים במקום אייקוני Lucide, padding לא אחיד בתגיות, badge "Made with Emergent" גלוי, וחוסר Empty States. תיקונים אלו הם "הקילומטר האחרון" שהופך את BrikOps ממוצר טוב למוצר שנראה premium.

## Tasks

### 1. החלפת אימוג'ים באייקוני Lucide
**קובץ:** שורת כפתורי Admin בעמוד פרויקט (כנראה `ProjectSettings.tsx` או `ProjectTabs.tsx`)

החלפות:
- 👥 צוות → `<Users />` מ-lucide-react
- 🏢 קבלנים וחברות → `<Building2 />`
- 💳 מנוי ותשלום → `<CreditCard />`
- 📋 מאשרי בקרת ביצוע → `<ClipboardCheck />`
- 📝 תבנית בקרת ביצוע → `<FileEdit />`
- 🔑 תבנית מסירה → `<KeyRound />`

**VERIFY:** בכל הכפתורים בשורת ה-Admin, האייקונים הם Lucide ולא אימוג'ים. בדוק ב-Chrome (macOS) ו-Safari (iOS) שנראים זהים.

### 2. אחידות Padding בתגיות קטגוריה
**קובץ:** `client/src/components/ui/Badge.tsx` או קומפוננטת Tag
- אחד את כל התגיות (חשמל, קריטי, דלתות, ריצוף, בינוני) ל:
  - `px-2 py-0.5 rounded-full text-xs font-medium`
- ודא שהצבעים עקביים:
  - קריטי: `bg-red-100 text-red-700`
  - בינוני: `bg-amber-100 text-amber-700`
  - נמוך: `bg-blue-100 text-blue-700`
  - קטגוריה (חשמל, דלתות, ריצוף): `bg-slate-100 text-slate-700`

**VERIFY:** כל התגיות ברשימת ליקויים בגודל ורווח אחידים.

### 3. הסרת Badge "Made with Emergent"
**קובץ:** `client/src/App.tsx` או `Layout.tsx` או `index.html`
- מצא את הלינק/badge של Emergent בתחתית העמוד
- הסר או הסתר: `display: none` / הסרה מה-DOM
- אם זה חלק מ-Emergent SDK, בדוק אם יש הגדרה `showBadge: false`

**VERIFY:** אין שום branding של Emergent גלוי באף עמוד.

### 4. Empty State — עמוד פרויקטים
**קובץ:** `client/src/pages/Projects.tsx`
- הוסף מצב ריק כשאין פרויקטים:
  - אילוסטרציה/אייקון גדול (למשל Building2 מ-Lucide, 64px, slate-300)
  - כותרת: "אין עדיין פרויקטים"
  - תיאור: "צור פרויקט חדש כדי להתחיל לנהל ליקויים ומסירות"
  - כפתור CTA: "צור פרויקט ראשון" (amber, גדול)

**VERIFY:** הסר את כל הפרויקטים מחשבון הדמו (או צור משתמש חדש) ובדוק שה-empty state מופיע.

### 5. Placeholder חיפוש ספציפי
**קבצים:**
- חיפוש פרויקט: שנה מ-"חיפוש פרויקט..." ל-"חפש לפי שם, קוד או כתובת..."
- חיפוש דירה: שנה מ-"חיפוש דירה..." ל-"חפש לפי מספר דירה..."
- חיפוש ליקוי: שנה מ-"חיפוש ליקוי..." ל-"חפש לפי תיאור, קטגוריה או קבלן..."

**VERIFY:** כל שדות החיפוש מציגים placeholder ספציפי שעוזר למשתמש.

### 6. דשבורד — הסבר לציון פעילות צוות
**קובץ:** קומפוננטת TeamActivity בדשבורד
- הוסף tooltip על העיגול של ציון הפעילות (15) שמסביר: "ציון מבוסס על פעילות באפליקציה ב-7 הימים האחרונים. ציון מקסימלי: 100"
- אם אפשר, הוסף צבע לעיגול: אדום 0-30, צהוב 31-60, ירוק 61-100

**VERIFY:** hover על ציון הפעילות מציג tooltip מידעי.

## DO NOT
- אל תשנה את הפונט הראשי
- אל תשנה את מבנה הטאבים
- אל תוסיף אנימציות חדשות — רק תתקן עקביות
- אל תשנה את הלוגו או הצבע הראשי

## VERIFY
- כל האימוג'ים הוחלפו ב-Lucide icons
- כל התגיות בגודל אחיד
- Badge Emergent לא מופיע
- Empty state עובד
- בדוק ב-375px ו-1440px
