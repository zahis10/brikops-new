מערכת סינון רב-מימדית (חדש)

Bottom Sheet בסגנון BottomSheetSelect הקיים (פטרן מוכר של BrikOps). הסינון נשמר ב-URL params → יציב ברענון, מאפשר שיתוף קישור, ומוחל גם על הייצוא.

סינון

רב-בחירה
ארזי הנגב ✓ אלקטרה ב' ✓ דניה סיבוס שיכון ובינוי אחים דונייבסקי + 7 אחרים
בטיחות חשמל אינסטלציה גמר טפסנות ברזנות
עבודה בגובה פיגומים עבודת חשמל חפצים נופלים פתחים ברצפה/קיר מגדל הרמה חפירות + 3

7 ממדי סינון

ממומשים כ-Pydantic query params. כל ממד ⇒ index ב-MongoDB. כל שילוב ⇒ URL query string.

company_id[]
חברה / קבלן — רב-בחירה · reuse של project_companies הקיים
profession
מקצוע — תואם ל-tradeService הקיים
category
קטגוריה — 10 קטגוריות מתוקננות (seeded)
status
סטטוס — open / closed / overdue
severity[]
חומרה — 1 / 2 / 3 (אין 0)
date_from/to
טווח תאריכים — חיתוכי זמן מוגדרים מראש + Custom
flags[]
דגלים — has_photos / overdue / no_corrective_action / expiring_30d
GET /api/safety/{project_id}/documents?
  company_id=abc,def&
  category=height_work&
  severity=2,3&
  status=open&
  date_from=2026-03-22&
  flags=has_photos,overdue

פתרון התנגשות ארכיטקטונית: project_companies

בסקירה עמוקה הסתבר שקיים כבר ב-BrikOps project_companies (קבלן רמה-פרויקט) — עם שדות name, trade, contact_name, contact_phone, contact_email. במקום לבנות contractor_companies חדש ולייצר כפילות, נבצע הרחבה בלבד.

שדה קיים? Phase 1 הערה
name✓ קיים—ללא שינוי
trade✓ קיים—ללא שינוי
contact_name / phone / email✓ קיים—ללא שינוי
registry_number✗ חסר+ הוסף (optional)מספר קבלן רשום (רישוי קבלנים). חובה לדוחות רגולציה.
is_placeholder✗ חסר+ הוסף (default=false)מסמן חברה שנוצרה מהיר מתוך טופס הדרכה (לפני שיש registry_number)
safety_contact_id✗ חסר+ הוסף (optional)reference למשתמש ש-role=safety_officer של החברה
deleted_at / deleted_by / deletion_reason✗ חסר+ הוסף (soft-delete)נדרש ל-SOC2 + רגולציה 7 שנים · נוסף לכל הטבלה · migration ל-collection קיים
retention_until✗ חסר+ הוסף (auto = +7y)מחושב ב-insert. שימושי ל-TTL future-phase.

Migration Plan

  1. בסיום ההרחבה — 6 שדות חדשים מתווספים ל-schema של project_companies כ-optional
  2. לכל המסמכים הקיימים: { $set: { deleted_at: null, retention_until: created_at + 7y } }
  3. Index חדש: { project_id: 1, deleted_at: 1 } — לכל query קיים מתווסף פילטר deleted_at: null
  4. קיים companies_router.py → נוסיף 3 endpoints: DELETE (soft), POST /restore, GET /deleted (admin only)
  5. האפליקציה ממשיכה לעבוד על אותו טאב "קבלנים וחברות" הקיים — מחזירה רק מסמכים לא מחוקים
