🛡 SOC2 Readiness — כל החלטה מקודדת לקונטרול

TSC Control מה בנוי ב-Phase 1 איך זה נאכף
Security · Access Control (CC6.1) RBAC double-gated: role + project membership require_roles('project_manager', 'management_team') + _check_project_access() על כל endpoint
Security · Audit Logging (CC7.2) 100% audit coverage על כל mutation _audit(entity_type, entity_id, action, actor_id, payload) אחרי כל create/update/soft_delete
Confidentiality · PII (C1.1) ת.ז. מוצפנת בlist endpoints ובaudit logs mask_id_number() → ****1234. רק /workers/{id} מחזיר מלא ל-management
Availability · Change Mgmt (CC8.1) Feature flag off-by-default + kill switch ENABLE_SAFETY_MODULE=false בprod. הפעלה idempotent, ביטול מחזיר 404 מיידית
Processing Integrity · Retention (CC2.1) Soft-delete + retention_until = 7 שנים שדות deleted_at, deleted_by, deletion_reason, retention_until בכל 4 הcollections. אין hard delete.
Security · Separation of Duties (CC6.3) Cascading soft-delete + placeholder protection מחיקת חברה עם עובדים → 400. מחיקת עובד → cascade מחיקת הדרכותיו. "אין חברה" מוגן ממחיקה ועריכה.
Monitoring · Anomaly Detection (CC7.3) Audit events פר-project, פר-actor, פר-entity audit_events indexed by project_id + actor_id. cron עתידי יגלה patterns חשודים.
Regulatory · ישראל (תקנות בטיחות 1988) Training types seeded עם regulatory_source 8 טיפוסי הדרכה עם אזכור תקנה (למשל: "תקנות בטיחות בעבודה בגובה, התשס״ז-2007")
