P1 — WhatsApp Manual Rejection Notification
==================================================

T001: Backend — Manual QC rejection WhatsApp endpoint
- Blocked By: []
- Details:
  - New endpoint: POST /api/qc/run/{run_id}/stage/{stage_id}/notify-rejection
  - Accepts: { recipient_user_id, item_id (optional), message (optional — default template) }
  - RBAC: PM / approver / management_team only. Contractors blocked.
  - Logic:
    a) Validate run_id, stage_id exist and stage/item was actually rejected
       - If item_id provided: validate item exists in stage and item reviewer status is rejected (or equivalent rejected state)
       - If no item_id: validate stage is currently rejected
    b) Validate recipient_user_id is a member of the project (project_memberships)
    c) Fetch recipient phone_e164 from users collection — if missing, return 400 with clear Hebrew message
    d) Build message content:
       - project name
       - floor
       - stage title
       - item title (if item-level)
       - rejection reason (stage/item level as relevant)
       - direct link to stage/item in system
       - first photo URL if available
    e) Send via existing WhatsAppClient
       - text only OR template+image if current infra supports it
    f) Audit event on success: qc_rejection_whatsapp_sent_manual
       - payload: project_id, run_id, stage_id, item_id, recipient_user_id, recipient_role, has_image, result_status
    g) Audit event on failure as well (separate action or same action with result_status=failed)
  - Files: backend/contractor_ops/qc_router.py
- Acceptance:
  - 200 on success
  - 400 if no phone
  - 403 unauthorized
  - audit logged (success and failure paths)
