PHASE A — Manual WhatsApp from task detail (V1)

PHASE B — Automatic WhatsApp pipeline (V2)
1) Create collection `notification_jobs`:

2) Enqueue job on:
   - task created+assigned
   - status changed to waiting_verify
   - task closed
3) Add worker loop with retry policy (max 3 attempts, exponential backoff).
4) Add idempotency key per (task_id + event_type + assignee + minute bucket) to avoid duplicate sends.
5) Feature flag:
   WHATSAPP_ENABLED=true/false
   If false => dry-run only (log + mark skipped).
