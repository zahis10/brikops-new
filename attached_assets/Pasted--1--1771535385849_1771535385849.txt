מעולה, תודה. אני מאשר את הכיוון — רק כמה חידודים/דרישות כדי שלא ניפול לבאגים שוב:

1) שינוי תפקיד — UI חייב להתעדכן מיידית + להפריד “שם” מול “תפקיד”
- חשוב: “עוזר בטיחות” יכול להיות שם משתמש (display_name) וזה בסדר שלא משתנה.
- מה שחייב להשתנות מיד אחרי 200:
  א) ה-Role badge בפרופיל (“תפקיד בפרויקט”)  
  ב) הטבלה ב-/admin/users  
  ג) הרשימה תחת “פרויקטים (N)” ב-Drawer (התפקיד ליד הפרויקט)
- טכני: לא מספיק רק refetch user-details אם ה-routing/הרשאות נשענים על session.
  לכן צריך גם:
  - refresh ל-/auth/me (או endpoint מקביל) כדי לעדכן AuthContext/session
  - או להחזיר מהשרת response שמכיל את ה-role החדש ולבצע update מקומי + invalidate cache.
- בבקשה להוסיף Badge ברור בפרטי משתמש:
  “שם” / “תפקיד בפרויקט” / “platform_role” (רק תצוגה, לא לעריכה).

2) באג routing של PM — להגדיר כלל חד-משמעי
- כרגע PM נזרק ל-flow של קבלן. זה חייב תיקון.
- כלל שאני רוצה:
  - project_manager / management_team / viewer: נוחתים ב-/projects (רשימת פרויקטים)
    * אם יש בדיוק פרויקט אחד (trial לרוב) אפשר “המשך לפרויקט האחרון” אבל עדיין לא לזרוק למסך קבלן.
  - contractor: נוחת למסך המשימות שלו.
- אם אתם רוצים לשמור את מה שמופיע בתיעוד (“PM → /projects/:id/control”) — סבבה, אבל רק אם זה באמת דשבורד/שליטה של PM ולא דף קבלן.
- בבקשה לבדוק: role detection מגיע מהשרת נכון גם אחרי refresh (לא רק בלוגין).

3) UX מסך ליקויים — שיפור אינטואיטיביות במובייל
- כן. להוסיף:
  - כותרת/הסבר קצר
  - empty state עם CTA “הוסף ליקוי”
  - סידור כפתורי פעולה במובייל בצורה עקבית (שלא יצופו ויבלבלו)
  - ריווחים טובים בצ’יפים/פילטרים באייפון

4) מודול “תוכניות פרויקט” — החלטות מוצר + מבנה
- מסכים ליצור collection חדש project_plans (לא לערבב עם unit_plans).
- UX:
  - “תוכניות פרויקט” פותח עמוד ייעודי
  - Sidebar ימני של דיסציפלינות (אדריכלות/קונסטרוקציה/חשמל/אינסטלציה/פיתוח…)
  - כפתור “+ הוסף קטגוריה” (דיסציפלינה מותאמת לפרויקט)
  - בתוך קטגוריה: רשימת קבצים + “+ העלה תוכנית”
- הרשאות (MVP):
  - PM + management_team: add category + upload/delete plan
  - viewer: קריאה בלבד
  - contractor: קריאה בלבד לכל התכניות (כרגע הכי פשוט; פילטור לפי תחום אפשר בהמשך)
- Audit:
  - project_plan_upload / project_plan_delete
  - project_discipline_add (הוספת קטגוריה)
  - חובה note למחיקות/שינויים (כמו סטנדרט billing/admin)
