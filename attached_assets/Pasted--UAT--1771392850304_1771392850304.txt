סבבה, תודה. לפני שאני מאשר סופית – אני עושה UAT אצלי.
תשלח לי בבקשה הוכחות + איפה זה נוגע בקוד (קבצים/פונקציות) + דוגמאות API:

0) Version Lock (חובה)
	•	Git SHA סופי של הפריסה.
	•	איפה ה-SHA מוצג בפרונט (Footer / debug endpoint) ואיפה בבק.
	•	רשימת קבצים ששונו (frontend + backend).
	•	הוכחה שהפרונט והבק על אותו SHA (צילום מסך Footer + צילום של debug/version).

1) “הוסף חברה חדשה” – תחומים דיניים + חובה לכל השדות

איפה באפליקציה: פרויקט → טאבים → חברות → הוסף חברה חדשה
מה להוכיח:
	•	צילום מסך שה-dropdown “תחום” מציג את כל ה־14 trades בעברית.
	•	צילום מסך שכוכבית (*) ליד: שם חברה, תחום, איש קשר, טלפון.
	•	צילום מסך שכפתור “הוסף חברה” Disabled עד שממלאים את כולם.
	•	דוגמת Response אמיתית של GET /api/trades (להדביק JSON).
	•	דוגמת 422 בעברית כשחסר שדה (payload + response).
איפה בקוד: לציין את קבצי ה-UI של המודאל + ה-endpoint/validation בצד שרת.

2) חץ חזרה במסך ליקויים

איפה: פרויקט → ליקויים (הדף עם הרשימה וה-chips)
מה להוכיח:
	•	וידאו קצר: נכנס לליקויים → עושה Refresh → לוחץ חץ אחורה → חוזר תמיד ל־/projects/:projectId.
איפה בקוד: הקובץ של Tasks page header/navigation (איפה שהיה navigate(-1)).

3) נרמול טלפון בחברה

איפה: אותו מודאל “הוסף חברה חדשה”
מה להוכיח:
	•	הזנה של 0501234567 מתקבלת (לא מחייב +972).
	•	בדאטה נשמר canonical E.164 (אם זה מה שעשיתם) – תראה payload שנשלח + ערך שנשמר/חוזר.
איפה בקוד: פונקציית הנרמול + איפה היא נקראת ביצירת חברה.

4) Invite / הוסף איש צוות – חובה ברור

איפה: פרויקט → צוות → הוסף איש צוות
מה להוכיח:
	•	כוכבית/חובה ליד: טלפון, שם מלא, תפקיד, וגם trade כשנבחר Contractor.
	•	צילום מסך שהכפתור Disabled עד שממלאים.
איפה בקוד: הקומפוננטה של invite form + ה-validation בבק.

5) קומות – ביטול “הכנס קומה” ואיחוד לתוך “הוסף קומות”

איפה: פרויקט → מבנה/בניין
מה להוכיח:
	•	צילום מסך שאין יותר כפתור נפרד “הכנס קומה” בסרגל התחתון.
	•	צילום מסך שבמודאל “הוסף קומות” יש Toggle/מצב: טווח קומות וגם הכנסה לפי מיקום (מעל/מתחת).
איפה בקוד: ProjectControlPage / bottom bar + קומפוננטת AddFloorsModal.

בנוסף – למניעת “אצלי לא מתעדכן”

אם יש caching/service worker:
	•	תציין אם קיים, ואם כן איך לנקות (hard refresh / clear site data).
ואם אין – תציין שאין SW כדי שנדע שזה לא cache.
