1) חובה: “תחום” דינאמי ברמת פרויקט + הוספת קבלנים מתוך התחום

כרגע אני רואה רק את ה-base (ה-14 trades המובנים) ואין לי אפשרות להוסיף תחום חדש או להוסיף קבלן חדש מתוך התחום.
הדרישה שלי: אנחנו נותנים Base, אבל מנהל פרויקט חייב להיות דינאמי — הוא יוכל להוסיף תחומים/קבלנים שלא קיימים במאגר הכללי, וזה יישמר רק בפרויקט הספציפי.

מה אני רוצה בפועל (UX):

בכל dropdown של “תחום” (גם “הוסף חברה” וגם “הזמן קבלן/איש צוות”):
	•	בתחתית הרשימה יהיה: “+ הוסף תחום חדש”
	•	לחיצה → פותחת input קצר “שם תחום בעברית”
	•	Save → התחום מתווסף לרשימת התחומים של הפרויקט הזה בלבד ומופיע מיידית ב-dropdown
	•	אחרי זה אפשר להזמין קבלן/להוסיף חברה תחת התחום החדש

בנוסף, במסך שבו בוחרים “תחום” לקבלן (Invite Contractor):
	•	אם בחרתי תחום מסוים, תהיה לי גם אפשרות “+ הוסף קבלן חדש לתחום הזה” (CTA קטן ליד בחירת התחום / או בתוך הרשימה אחרי בחירה)
	•	זה פשוט פותח את מודאל ההזמנה עם role=contractor ו-trade כבר prefilled.

Backend (מינימום נדרש):
	•	Endpoint חדש (פרויקט-סקופ):
	•	GET /api/projects/{projectId}/trades → מחזיר merged של base(14) + custom trades של הפרויקט
	•	POST /api/projects/{projectId}/trades → יצירת תחום חדש לפרויקט (label_he) + יצירת key יציב
	•	RBAC: רק Admin/PM יכולים ליצור תחום חדש לפרויקט (Management/Contractor לא).
	•	ולידציה:
	•	תחום באותו שם/מפתח בפרויקט → 409
	•	לא לשנות/לגעת ב-global trades הקיימים.

איפה זה נוגע בקוד (צד לקוח):
	•	frontend/src/components/ManagementPanel.js
	•	AddCompanyForm: trade dropdown צריך להביא מ-/api/projects/{id}/trades ולא רק base
	•	Invite forms: גם הם צריכים להשתמש באותו endpoint כדי לראות תחומים מותאמים לפרויקט
	•	frontend/src/pages/ProjectTasksPage.js (או קומפוננטת chips)
	•	אם יש “הוסף קבלן” – שיפתח את מודאל ההזמנה עם prefill

Acceptance:
	•	אני יכול להוסיף תחום חדש בעברית בפרויקט → והוא מופיע מיד ב-dropdownים.
	•	אני יכול להזמין קבלן תחת תחום חדש → והוא מופיע בצ’יפים/ב-buckets של הפרויקט.
	•	שום דבר לא “ננעל” ל-14 תחומים בלבד.
