M4.8 — מסך “הפרויקטים שלי” אחרי התחברות + ניווט לפרויקט

הבעיה כיום

אחרי Login המשתמש נוחת ישר למסך הראשי הקיים, והפרויקטים “יושבים למטה” ולא נגישים. אנחנו רוצים UX ברור: קודם בוחרים פרויקט → ואז עובדים בתוך הפרויקט.

⸻

1) Flow חדש אחרי התחברות

מה צריך לקרות
	•	אחרי התחברות (כולל “כניסה מהירה לדמו”) המשתמש מגיע ל־/projects (או /my-projects) עם מסך “הפרויקטים שלי”.
	•	המסך מציג רק פרויקטים שהמשתמש חבר בהם (לפי RBAC).
	•	לחיצה על פרויקט מעבירה ל־/projects/:projectId (המסך הקיים שלכם היום – ProjectControl / Overview).

UI רצוי למסך פרויקטים
	•	כרטיס לכל פרויקט עם:
	•	שם פרויקט
	•	קוד/מספר (אם יש)
	•	כתובת (אם קיימת)
	•	תצוגה יפה ונגישה למובייל (Grid 1–2 עמודות, כמו “cemento” בתמונה)
	•	חיפוש פרויקט (אופציונלי אבל מומלץ) + מצב ריק (“אין פרויקטים עדיין”).
	•	לחצן “+” ליצירת פרויקט (רק Admin/PM) – אם כבר קיים אצלכם במקום אחר, אפשר לשים פה קיצור.

Acceptance
	•	אחרי Login תמיד מגיעים למסך פרויקטים (ולא נוחתים ישר בתוך פרויקט).
	•	לחיצה על פרויקט פותחת את המסך הקיים /projects/:id.
	•	כפתור Back מתוך פרויקט מחזיר ל־/projects (רשימת הפרויקטים).

⸻

2) “פרויקט אחרון” (נוחות)

מה להוסיף
	•	אם יש למשתמש “פרויקט אחרון” (last_selected_project_id):
	•	להראות בראש המסך “המשך לפרויקט האחרון” + כפתור “היכנס”.
	•	אבל לא לדלג אוטומטית (כדי לא לבלבל).
	•	לשמור בחירה של פרויקט ב־localStorage (client) או בפרופיל משתמש (server) אם אתם רוצים שזה יעבור בין מכשירים.

⸻

3) ניווט / Routing

מה לשנות
	•	Redirect אחרי login:
	•	כיום כנראה מפנה ל־/ או לדשבורד.
	•	לשנות ל־/projects.
	•	Route guards:
	•	אם משתמש נכנס ל־/projects/:projectId בלי הרשאה לפרויקט → 403/הודעה בעברית + כפתור חזרה לרשימת פרויקטים.

⸻

4) הוכחות + בדיקות (חובה לפני “סגור”)

Proof/Deliverables
	1.	צילום מסך של מסך “הפרויקטים שלי” אחרי Login (אייפון).
	2.	וידאו קצר: Login → רואים Projects → לחיצה על פרויקט → נכנסים למסך הפרויקט הקיים → Back חוזר ל־Projects.
	3.	דוגמת API:
	•	GET /api/projects?mine=1 (או endpoint קיים) + response עם 2–3 פרויקטים.
	4.	Git SHA סופי + רשימת קבצים שנגעו.

Tests
	•	Backend: test ש־user מקבל רק את הפרויקטים שהוא member בהם.
	•	Frontend: basic navigation test / manual checklist:
	•	login → /projects
	•	click project → /projects/:id
	•	back → /projects

⸻

עוד המלצות (שווה להכניס עכשיו כי זה “אותו אזור”)
	1.	Project switcher קטן בתוך מסך פרויקט (בכותרת): לחיצה → חוזר לרשימת פרויקטים או פותח dropdown “החלף פרויקט”.
	2.	Pinned projects / מועדפים (כוכב) לפרויקטים חשובים.
	3.	סינון לפי סטטוס פרויקט (פעיל/ארכיון) אם זה רלוונטי בעתיד.
