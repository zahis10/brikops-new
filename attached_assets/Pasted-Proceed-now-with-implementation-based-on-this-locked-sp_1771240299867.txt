## 1) Goal
Build BedekPro for professional inspectors (not tenant-self-report flow), with Reporto-like structured findings and a professional Hebrew RTL PDF.

## 2) Required finding structure (must exist in UI + DB + PDF)
Each finding must include:
- finding_number (e.g., 1.1)
- category (professional categories only)
- title/defect_description (ממצא)
- location (מיקום)
- note (הערה) [optional]
- recommendation (המלצה)
- standard_reference (תקן) – free text + suggested standards list
- pricing:
  - unit_price
  - quantity
  - unit_label (e.g., קומפ', מ"ר, יח')
  - total_price (calculated)
- images[] (1..N)

## 3) Professional categories (seed list)

## 4) Property details section (inspection-level)
- property_type
- project/building/floor/apartment
- client_name
- occupancy (מאוכלס כן/לא)
- utility connections (חשמל/מים/גז)
- room list

## 5) Expert profile (reusable)

- full_name
- certification/education
- years_experience
- license/registration (if available)
- phone/email/address
- signature image
- logo

  1) cover/header section
  2) property details
  3) standards/methodology section
  4) findings pages
