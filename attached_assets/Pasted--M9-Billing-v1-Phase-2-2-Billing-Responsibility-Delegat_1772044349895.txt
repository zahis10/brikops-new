M9 Billing v1 — Phase 2.2 (Billing Responsibility Delegation UX + Hebrew Cleanup)
Objective
לסגור את פער ה-UX שנותר אחרי Phase 2.1:
1) להבהיר למשתמשים מי אחראי על החיוב בארגון ואיך מעבירים אחריות
2) לאפשר הקצאת אחריות חיוב מתוך UI ברור (בלי בלבול)
3) להשלים ניקוי טקסטים/לייבלים באנגלית שנשארו במסכי Billing/Admin
4) בלי לשנות לוגיקת תמחור / snapshots / חישובים עסקיים

Scope (חשוב)
- כן: UX + זרימת הקצאת אחריות + חיבור למסך/ניהול תפקידי ארגון
- לא: שינוי תמחור, שינוי tiers, שינוי state machine, שינוי paywall, שינוי billing calculations

Current Pain (מה המשתמש רואה)
- לא ברור איך “מעבירים חיוב למישהו אחר בארגון”
- יש טקסטים באנגלית/technical IDs במסכים
- הזרימה קיימת חלקית (PM handoff / org roles), אבל לא ברורה מספיק בתוך ה-UI

Quality / Release Requirements (Mandatory — High/Sensitive: Billing)
1) No closure without proof
- אין לסגור על בסיס "נראה תקין"
- חובה: root cause (אם היה באג), fix, regression checks, API/UI proof

2) Guardrails on sensitive flows
- Billing/UI:
  - explicit RBAC checks (backend source of truth)
  - frontend does not assume permissions
  - clear disabled/hidden states למי שלא מורשה
- Org role changes:
  - strict org scoping
  - audit event on every role change

3) No “user confusion” closure without evidence
- "לא הבין את הזרימה" יכול להיות symptom בלבד
- חובה לספק UX proof ברור (screenshots + flow verification)

4) Release gates before customer-facing close
- Smoke tests (owner / PM / billing_admin / viewer)
- Negative tests (unauthorized role updates, cross-org access)
- Attach evidence before marking resolved
