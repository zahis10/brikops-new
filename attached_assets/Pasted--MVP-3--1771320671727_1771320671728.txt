ספרינט חדש (MVP מהיר, 3 ימים): דף דירה ייעודי + פתיחת ליקוי מתוך דירה + שליחת וואטסאפ עם תמונה

המטרה:
מהמסך של המבנה, בלחיצה על מספר דירה (למשל 38), לעבור לדף/חלון של הדירה הספציפית בלבד, עם כל הליקויים שלה, ואפשרות לפתוח ליקוי חדש שמקושר אוטומטית לדירה הזו.
בעת יצירת ליקוי — לשלוח לאיש הקשר בוואטסאפ את פרטי הליקוי + בניין/קומה/דירה + תמונה.

⸻

1) Scope מדויק

A. ניווט לדירת יחידה
	•	בלחיצה על chip של דירה במסך המבנה → ניווט ל:
	•	/projects/:projectId/units/:unitId
(או פתיחת drawer/full-screen sheet מובייל, העיקר שזה דף ייעודי לדירה)
	•	בדף הדירה להציג:
	•	כותרת: פרויקט / בניין / קומה / דירה
	•	KPI קטן: פתוחות / בטיפול / סגורות (של הדירה בלבד)
	•	רשימת ליקויים של הדירה בלבד
	•	כפתור קבוע: “ליקוי חדש לדירה”

B. פתיחת ליקוי מתוך הדירה
	•	פתיחת ליקוי מתוך דף דירה עם prefill/lock לשדות:
	•	project_id, building_id, floor_id, unit_id
	•	שדות למילוי:
	•	קטגוריה
	•	תיאור
	•	שיוך לקבלן/חברה/איש קשר
	•	תמונות (לפחות 1, אפשר כמה)
	•	עדיפות
	•	אין צורך לבחור שוב פרויקט/בניין/קומה/דירה כשפותחים מתוך הדירה.

C. וואטסאפ אוטומטי עם תמונה
בעת שמירת ליקוי:
	•	יצירת notification job אוטומטי (לא חסימת UX)
	•	שליחה לוואטסאפ של איש הקשר שנבחר:
	•	פרויקט, בניין, קומה, דירה
	•	מזהה ליקוי
	•	תיאור
	•	קטגוריה
	•	לינק ישיר לליקוי
	•	תמונה (לפחות הראשונה כ-image message; נוספות כקישורים/הודעות נוספות)
	•	אם אין מספר טלפון תקין → חיווי ברור + לוג audit.
	•	להשתמש במנגנון הקיים של retries/idempotency/webhook statuses.

⸻

2) RBAC (לא לשבור מודל קיים)
	•	Owner/Admin/PM: גישה מלאה לדף דירה + פתיחת ליקוי + שליחת וואטסאפ.
	•	PM: רק בפרויקטים שהוא חבר בהם.
	•	Contractor/Viewer:
	•	אם לא הוגדר אחרת: קריאה בלבד, בלי פעולות ניהול.
	•	לאפשר פתיחת ליקוי רק אם זה חלק מההרשאות הקיימות (לא להרחיב הרשאות בלי אישור).

⸻

3) UX מובייל RTL (חשוב)
	•	RTL מלא, מובייל-first.
	•	כפתור “ליקוי חדש לדירה” sticky בתחתית.
	•	ליסט ליקויים עם פילטר מהיר (סטטוס/קטגוריה).
	•	שגיאות ברורות בעברית.
	•	מהירות: optimistic refresh אחרי יצירת ליקוי.

⸻

4) שיפורי ייעול (אם נכנס בזמן)
	1.	תבנית הודעת וואטסאפ אחידה עם פורמט קריא וקבוע.
	2.	Batch notify: אם יש כמה אנשי קשר לקבלן — לבחור “אחד” או “כולם”.
	3.	Quick actions בדף דירה:
	•	“שכפל ליקוי מדירה אחרת”
	•	“פתח WhatsApp ידני”
	4.	חיווי משלוח: timeline קטן בדף הדירה (נשלח/נמסר/נכשל + retry למנהל).

⸻

5) Definition of Done (DoD)

לא לסגור ספרינט בלי כל אלה:
	1.	לחיצה על דירה מהמבנה פותחת דף דירה נכון (פרויקט/בניין/קומה/דירה תקינים).
	2.	יצירת ליקוי מתוך דף דירה יוצרת ליקוי מקושר לדירה בלבד.
	3.	נשלחת הודעת וואטסאפ עם כל הפרטים + תמונה ראשית בפועל (או dry-run מתועד אם feature flag כבוי).
	4.	PM רואה ומנהל רק פרויקטים שהוא חבר בהם (נא להציג הוכחה).
	5.	Contractor/Viewer לא מקבלים יכולות ניהול לא רצויות.
	6.	0 רגרסיות בפיצ’רים קיימים.
	7.	הוכחות מסירה:
	•	6–8 צילומי מסך אמיתיים מה-flow
	•	דוגמאות request/response
	•	דוגמת payload וואטסאפ
	•	תוצאות טסטים
	•	git SHA סופי

⸻

6) תוצר סופי שאני מצפה לקבל ממך
	•	רשימת endpoints שנוספו/עודכנו
	•	רשימת קומפוננטות UI שנוספו
	•	מה נשלח בדיוק בוואטסאפ (payload לדוגמה)
	•	סטטוס טסטים
	•	קישורי מסכים/צילומים
	•	git SHA
