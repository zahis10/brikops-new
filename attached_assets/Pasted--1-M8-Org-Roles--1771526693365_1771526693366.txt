היי, ממשיכים לשלב 1 (M8) — בבקשה להתקדם בזהירות, בלי “לערבב” שוב את כל נושא ה-Org/Roles שייצבנו עכשיו. חשוב: לא לבצע שינויי schema/מיגרציות נוספות ל-organization/role מעבר למה שחייבים לפיצ’רים הבאים. כל שינוי RBAC חייב להגיע עם בדיקות רגרסיה.

A) תיקון קריטי: Trial Banner / Paywall בלוגין
- כרגע אני רואה “תקופת ניסיון הסתיימה” עוד לפני שהתחברתי.
- דרישה: /login חייב להיות נקי לחלוטין (אין באנר, אין מודאל תשלום) כל עוד אין session פעיל.
- באנר/מודאל יופיעו רק אחרי התחברות (כשיש user/auth session), ורק אם billing status רלוונטי.
- נא להוסיף בדיקת regression: טעינת /login ללא session => 0 banners/modals.

B) ניקוי כפתורי דמו + הסבר רולים
- להסיר PM דמו כפול — להשאיר PM אחד בלבד.
- Viewer: להסדיר ולהבהיר מה הוא עושה:
  * אם משאירים Viewer: הוא חייב להיות Read-Only אמיתי (לראות דשבורד/ליקויים/תוכניות, בלי שום פעולה יוצרת/עריכה/שיוך/אישור/דחייה/העלאות).
  * להוסיף בדיקות שמוודאות ש-viewer חסום מכל write endpoints (403 ברור + הודעה בעברית).
- כפתורי דמו שצריך לי לצורך ניסויים: 
  1) Super Admin (dev בלבד)
  2) מנהל פרויקט (PM)
  3) צוות ניהולי (management_team)
  4) קבלן (contractor)
  5) צופה (viewer) — רק אם אנחנו באמת משאירים את הרול הזה.

C) מודול “Users Admin” (Super Admin בלבד)
- מסך חדש: /admin/users — נגיש רק ל-platform_role=super_admin.
- טבלה עם חיפוש: שם, טלפון, email (אם קיים), created_at, org_id, סטטוס מנוי (trial/active/suspended/comped), תאריך סיום trial, ומספר פרויקטים.
- Drawer/Details למשתמש: org memberships, project memberships (תפקידים פרויקטליים), ועוד.
- כל פעולה מנהלית חייבת audit event עם note חובה.

D) החלפת מספר טלפון / שחזור גישה
1) Self-serve (משתמש מחובר):
- Flow: המשתמש מזין מספר חדש -> OTP אימות למספר החדש -> update ל-phone_e164.
- דרישות: 
  * חסימת התנגשות: אם phone כבר קיים אצל משתמש אחר => 409 + הודעה בעברית.
  * אחרי שינוי מספר: session נשאר תקין או מבצעים re-auth מסודר (מה שיותר בטוח) — אבל לא להשאיר מצב “חצי מחובר”.
2) Admin recovery (super_admin):
- מתוך /admin/users: שינוי מספר למשתמש עם note חובה + אותו conflict check.
- זה מיועד למקרים שמשתמש איבד גישה למספר הקודם.

Acceptance / UAT קצר (חובה):
1) לפתוח /login (ללא התחברות) => אין באנר/מודאל.
2) להתחבר כ-PM trial => באנר trial מופיע רק אחרי login ומציג ימים נכון.
3) להתחבר כ-viewer => אפשר לראות, אבל כל ניסיון ליצור/לעדכן/להעלות => חסום (403) עם הודעה ברורה.
4) super_admin נכנס ל-/admin/users => רואה רשימה, חיפוש עובד, Drawer עובד.
5) שינוי מספר עצמי עובד עם OTP + אין כפילויות.
6) שינוי מספר ע"י super_admin עובד עם note + audit.
