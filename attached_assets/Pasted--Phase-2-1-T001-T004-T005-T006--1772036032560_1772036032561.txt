מעולה. מאשר להתקדם עם התוכנית Phase 2.1 כמו שכתובה:
T001–T004 במקביל → T005 → T006

כמה חידודים קטנים להוסיף כדי לסגור את זה מדויק:

1) T001 — Plan selector (מובייל/iPhone)
- לוודא שהסלקטור עובד גם באייפון (פתיחה/בחירה/שמירה)
- אם יש אצלנו MobileSelect/קומפוננטת מובייל מותאמת — להשתמש בה גם כאן (כדי לא ליפול שוב על בעיית select ב-iOS)
- קבלה: PM / org owner / org_admin / billing_admin / management_team רואים בסיסי / מקצועי / XL

2) T002 — שדה יחידות חוזיות
- state כמחרוזת (כמו שכתבתם)
- trim לפני parse
- ולידציה בשמירה: מספר שלם >= 1 בלבד
- ריק / 0 / ערך לא תקין → הודעת שגיאה ברורה
- להוסיף disabled לכפתור "שמור" בזמן request (מניעת לחיצות כפולות)
- inputMode="numeric" + pattern="[0-9]*" + placeholder "הזן מספר יחידות"

3) T004 — UX בקשת העברה
- CTA ברור: "בקש העברה לחיוב הארגון"
- בפתיחה להציג כותרת: "הערה לצוות החיוב (אופציונלי)"
- להוסיף שורת הסבר קצרה ל-PM:
  "לאחר שליחת הבקשה, צוות החיוב הארגוני יוכל לבחור תוכנית ולהשלים את ההגדרה."
- אחרי שליחה: toast הצלחה + קיפול + רענון
- חשוב: אם user_can_edit_billing=true לא להציג בכלל את כפתור בקשת ההעברה (מניעת כפילות/בלבול)
- חשוב: להציג את ה-CTA רק כש-setup_state=trial (לא במצבים אחרים)

4) T005 — AdminBillingPage מובייל
- מובייל-ראשון, מסודר ונקי
- KPI עליונים: סה"כ ארגונים / ארגונים פעילים / סה"כ הכנסה חודשית
- סקשן ארגונים מתקפל
- תוכניות תמחור מתקפל
- מיגרציה מתקפל (כבר קיים)
- בכרטיסי תוכניות: להציג tiers בטבלה ורטיקלית (קוד | תווית | מחיר) במקום chips מפוזרים
- לשנות כותרת ל-"תוכניות תמחור" (בלי Billing v1 באנגלית)
- יומן פעולות במובייל: כרטיסים מוערמים (לא טבלה)
- להוסיף תוויות עברית לפעולות audit דרך actionLabels.js:
  * project_billing_updated → עדכון חיוב פרויקט
  * contracted_units_changed → שינוי יחידות חוזיות
  * billing_handoff_requested → בקשת העברה לחיוב
  * billing_handoff_acknowledged → אישור קבלת בקשה
  * project_billing_setup_completed → השלמת הגדרת חיוב

5) כללי
- אין שינוי לוגיקה עסקית / RBAC / חישובים / snapshots
- הכל מאחורי BILLING_V1_ENABLED בלבד

אחרי סיום T006 תשלחו בבקשה 4 צילומי מסך:
1. PM — סלקטור תוכניות פתוח עם אפשרויות
2. PM — שדה יחידות חוזיות (ניקוי + הקלדה מחדש)
3. PM — בקשת העברה (סגור + פתוח)
4. Admin Billing — מובייל מסודר (viewport ~400px)
