היי, יש לנו 3 שדרוגים חשובים לליקויים + מסך ליקוי לקבלן. בבקשה לבצע הכל יחד:

1) פילטר סטטוס (צ’יפים) — לפי תפקיד

מטרה: קבלן לא יתבלבל ויראה רק מה שרלוונטי, ומנהלים יראו הכל.

Backend
	•	כרגע יש תמיכה ב־status=...
	•	להוסיף גם status_in=... (רשימה / multi-status) כדי לתמוך ב־“פעילים”.
	•	“פעילים” לקבלן = קומבינציה של כמה סטטוסים (לפי מה שקיים אצלנו):
לדוגמה: open + assigned + in_progress + pending_contractor_proof
(אם הסטטוסים אצלנו שונים — להתאים ל־keys הקיימים).

Frontend (ProjectTasksPage)
	•	להציג צ’יפים במקום/בנוסף לדרופדאון סטטוס:
	•	לקבלן: פעילים (ברירת מחדל) | ממתין לאישור מנהל | סגור
	•	למנהלים (Owner/Admin/PM/צוות ניהולי): הכל | כל סטטוס בנפרד (פתוח/בביצוע/ממתין לאישור מנהל/סגור/וכו’)
	•	חשוב: לשמור את ההקשחה הקיימת לקבלן (שלא יוכל לראות ליקויים שלא שלו) — הסטטוס רק מצמצם, לא “פותח”.

⸻

2) מיון ברירת מחדל לפי דחיפות (קריטי ראשון)

מטרה: אם יש ליקוי קריטי הוא תמיד ראשון למעלה.

Backend
	•	לשנות מיון ברירת מחדל מ־created_at desc ל:
	1.	priority/urgency: critical → high → medium → low
	2.	ואז updated_at desc
	•	להשתמש במפה מספרית (priority sort map) כדי ש־Mongo ידע למיין נכון.

⸻

3) קבלן יכול להעלות יותר מתמונה אחת כ”הוכחת תיקון”

מטרה: כשהקבלן נכנס לליקוי, תהיה לו פעולה ברורה של העלאת כמה תמונות תיקון ושליחה לאישור מנהל.

Backend
	•	לשדרג את POST /tasks/:id/contractor-proof כך שיתמוך:
	•	files: List[UploadFile] (multi)
	•	ועדיין להשאיר תאימות ל־single file אם נשלח רק קובץ אחד (backward compatible).
	•	להחזיר בתגובה proof_urls: [] (מערך).
	•	לשמור: שינוי סטטוס ל־pending_manager_approval + audit כמו היום.

Frontend (TaskDetailPage)
	•	להוסיף CTA בולט לקבלן:
	•	כפתור: “העלה תמונות תיקון”
	•	Preview לכל תמונה + Remove
	•	הוספה “אחת-אחת” (מומלץ למובייל)
	•	כפתור סיום: “שלח לאישור מנהל”
	•	אחרי הצלחה: toast + חזרה לרשימת הליקויים/עדכון מסך.

⸻

בדיקות + הוכחות (כמו שאנחנו עובדים תמיד)
	•	להריץ טסטים מלאים ולהביא פלט: X passed, Y skipped, 0 failures
	•	Version lock: git HEAD = version בצד שרת = footer
	•	UAT קצר:
	1.	קבלן: רואה ברירת מחדל “פעילים”, ומסוגל לעבור ל“ממתין לאישור מנהל”/“סגור”
	2.	קריטי מופיע ראשון
	3.	קבלן מעלה 2–3 תמונות, שולח לאישור, סטטוס עובר ל“ממתין לאישור מנהל”, והמנהל רואה את התמונות.
