ספרינט M3 — Israeli Phone Normalization (Invite + Auth)

מטרה:
לאפשר למשתמש להזין טלפון ישראלי בפורמט מקומי (ללא +972), 
ובשרת לנרמל אוטומטית ל-E.164 לשמירה ושליחה.

עקרון מחייב:
- UI מקומי ונוח לישראל (ללא חובה ל+972).
- Backend תמיד canonical: phone_e164 (לדוגמה: +972507569991).
- כל לוגיקת התאמה/חיפוש/דדופליקציה/שליחה תתבסס על phone_e164 בלבד.

1) Scope (Backend חובה)
A. פונקציית נרמול מרכזית אחת (shared utility):
- קלטים נתמכים:
  - 0507569991
  - 507569991
  - +972507569991
  - 972507569991
  - עם רווחים/מקפים/סוגריים
- פלט:
  - +972507569991
- כל קלט לא תקין -> 422 עם הודעה ידידותית בעברית.

B. נקודות חובה לשימוש בנרמול:
- יצירת הזמנה (invites)
- רישום משתמש
- התחברות/OTP לפי טלפון
- כל endpoint שמקבל/מעדכן phone

C. שמירת נתונים:
- phone_e164 (canonical, חובה)
- phone_raw (מה שהוקלד, אופציונלי לתחקור)
- phone_local_display (אופציונלי לתצוגה, לדוגמה 050-756-9991)

D. דדופליקציה והתאמה:
- כל בדיקות duplicate invites / user matching / auto-link מבוססות phone_e164
- אינדקס מתאים למניעת כפילויות פעילות להזמנות.

E. מגבלת invite:
- בהזמנות עם WhatsApp/OTP דרוש מספר נייד ישראלי בלבד (סלולר).
- מספר קווי/לא ישראלי -> 422 עם הסבר ברור.

2) Scope (Frontend UX)
A. שדות טלפון:
- placeholder: 05X-XXXXXXX
- keyboard: tel
- המשתמש יכול להזין מקומי בלבד (ללא +972)
- הודעת שגיאה ידידותית: "יש להזין מספר נייד ישראלי תקין"

B. התנהגות:
- תצוגה מקומית במסכים (למשל 050-756-9991)
- אין דרישה מהמשתמש להקליד +972
- השליחה לשרת יכולה להיות raw; השרת ינרמל authoritative

3) Migration/Backfill (חובה)
- סקריפט חד-פעמי לנרמול מספרים קיימים ב-users/invites/memberships (אם רלוונטי)
- דוח לפני/אחרי: כמה עודכנו, כמה נכשלו ולמה.

4) RBAC
- ללא שינוי הרשאות. שינוי טלפונים/הזמנות ממשיך לפי RBAC הקיים.

5) Definition of Done
1. Invite עם "0507569991" נוצר בהצלחה ונשמר כ-"+972507569991".
2. Register עם אותו טלפון (גם אם מוזן כ-507569991) עושה auto-link להזמנה.
3. Duplicate invite נחסם גם אם פעם אחת הוזן מקומי ופעם אחת בינלאומי.
4. מספר לא תקין מחזיר 422 עם הודעה בעברית.
5. אין רגרסיות ב-auth/invites/notifications.
6. WhatsApp payload משתמש ב-target_phone בפורמט E.164.

6) Deliverables
- PHONE_NORMALIZATION_PROOF.md
- LIVE_PHONE_NORMALIZATION_OUTPUT.txt
- רשימת endpoints שהושפעו
- דוגמאות request/response אמיתיות:
  - create invite (local input)
  - register (different input format, same number)
  - duplicate blocked
  - invalid phone 422
- Git SHA סופי
- ספירת טסטים + כמה חדשים נוספו
