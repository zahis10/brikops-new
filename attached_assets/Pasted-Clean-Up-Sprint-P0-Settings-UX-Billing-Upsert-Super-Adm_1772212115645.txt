T003 – Backend: Super Admin reset password endpoint
- Endpoint חדש: POST /api/admin/users/{user_id}/reset-password
- הגנה: JWT + require_stepup + בדיקת super_admin אמיתית מה-DB.
- Body: { new_password, note } (שניהם חובה)
- Validation: כמו בכל המערכת (מינימום 8, אות+ספרה, common-passwords blocked).
- Update:
  - set password_hash=bcrypt(new_password)
  - $unset password (אם קיים legacy)
  - session_version += 1 (invalidate sessions)
- Audit event:
  - action=admin_password_reset
  - actor_id (מי ביצע)
  - entity_id=user_id (על מי)
  - payload כולל note בלבד (לא לרשום סיסמה!)
Acceptance:
- SA מבצע reset → המשתמש מצליח להתחבר עם הסיסמה החדשה.
- טוקנים ישנים של המשתמש נחסמים (Session revoked).
- non-SA מקבל 403.
