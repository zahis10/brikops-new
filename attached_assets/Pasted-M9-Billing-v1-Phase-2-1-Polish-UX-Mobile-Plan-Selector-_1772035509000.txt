יש 5 תיקונים לפני מעבר לשלב 3. זה לא שינוי לוגיקה עסקית, אלא UX/RBAC display polish.

P0-1) Fix plan selector in ProjectBillingCard (dropdown shows no options)
Problem:
- "בחר תוכנית" נפתח בלי אפשרויות.
Likely cause:
- ProjectBillingCard fetches plans from /api/admin/billing/plans (SA-only), so org/PM users get empty list.
Required fix:
- Add new endpoint: GET /api/billing/plans/active (feature-flagged with BILLING_V1_ENABLED)
- Return active plans only (id, name, version, project_fee_monthly, tiers)
- Allow roles: SA, org owner, org_admin, billing_admin, PM, management_team (contractor/viewer optional deny)
- Update ProjectBillingCard to fetch plans from this endpoint (NOT admin endpoint)
Acceptance:
- Dropdown shows בסיסי / מקצועי / XL with all tiers on mobile and desktop.

P0-2) Fix contracted_units input UX (cannot delete default 0)
Problem:
- Numeric field is controlled with 0; user cannot clear it smoothly on mobile.
Required fix:
- Use string state for edit input (allow "")
- Parse/validate only on Save (integer >= 1)
- Add inputMode="numeric"
- Placeholder instead of hardcoded 0
Acceptance:
- User can clear field, type new number, save successfully.
