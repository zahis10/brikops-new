M9 Billing v1 — Phase 2.3 (Plan Merchandising UX + Value Communication)
Objective
להפוך את בחירת התוכנית (בסיסי / מקצועי / XL) מ"שדה טכני" לחוויית בחירה ברורה ומזמינה:
• להסביר ללקוח על מה הוא משלם
• להציג מה מקבלים בכל תוכנית
• להדגיש הבדלים בין התוכניות
• להציג פירוק מחיר ברור (תוכנית + מדרגת יחידות = סה״כ)
• בלי שינויי לוגיקה עסקית / תמחור / RBAC / snapshots / paywall

Scope (חשוב)
• Phase 2.3 הוא UX + תוכן בלבד (Merchandising / Presentation)
• אין שינוי בחישוב monthly_total
• אין שינוי ב-plan_id / tier_code / snapshots
• אין feature gating חדש בין תוכניות (רק הצגה/הסבר)
• הכל מאחורי BILLING_V1_ENABLED

Core Principle (כדי לא להטעות לקוחות)
אם כרגע אין הבדלים פונקציונליים אמיתיים באפליקציה בין Basic/Pro/XL:
• נציג את ההבדל כ"התאמה להיקף עבודה/מורכבות/סדר גודל"
• לא נבטיח פיצ'רים שלא קיימים בפועל
• נשתמש בשפה אמינה: "מתאים ל...", "מיועד ל...", "מומלץ עבור..."

Deliverables
1) כרטיסי תוכניות ברורים (תוכן + תמחור)
2) פאנל "מה מקבלים" כשבוחרים תוכנית
3) פירוק מחיר ברור לפרויקט (תוכנית + מדרגת יחידות)
4) תוויות/קופי שיווקי בעברית
5) הדגשת תוכנית מומלצת (מקצועי)
6) בדיקות UI + צילומי מסך
