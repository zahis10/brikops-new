## 2. איפה אנחנו מול Apple App Store Review?

### מצב נוכחי
- ✅ **TestFlight Internal Testing** — עובד, Build 4 זמין
- ❌ **Apple Sign-In** — לא הוגש ל-review (משימה #30, #31 in_progress)
- ❌ **External Reviewers** — לא מוגדרים
- ❓ **App Store Connect metadata** — לא אומת בסשן הזה (תיאור, keywords, screenshots)
- ❌ **Production submission** — לא הוגש

### מה חסר להגשה ל-App Store Review
1. Apple Sign-In מלא ותקין (פאזה 5 — **משימה #30, #31 פתוחות**)
2. Screenshots ב-5 מידות שונות (iPhone 6.7", 6.5", 5.5", iPad Pro 12.9", iPad Pro 11") — **משימה #32 סגורה, הקבצים קיימים**
3. App description + keywords + support URL + privacy policy URL
4. Age rating questionnaire
5. Pricing + availability
6. Submit for Review — Manual click

### המשוכה הקריטית
Apple Sign-In — אי אפשר להגיש ל-review בלי זה (Apple דורש אם יש social login אחרים, ויש לך Google Sign-In). זו **המשימה שצריכה להיסגר ראשונה** לפני submission.

## 5. Wave 5 — מה לעשות עכשיו (ביצוע מיידי)

### מה Wave 5 מתקן
1. iOS swipe-back gesture (capacitor.config.json)
2. Android hardware back button (listener ב-App.js + `@capacitor/app`)
3. Android launcher icon safe-zone (5 קבצי mipmap)
4. PDF/Excel export native (share sheet במקום `<a download>`)
5. Version bumps: iOS 1.0.5/5, Android 1.0.10/10

### הסדר המדויק

**צעד 1 — ברפליט:**
```
./deploy.sh --prod
```
(דוחף ל-GitHub + מעדכן Capgo OTA)

**צעד 2 — במק:**
```bash
cd ~/brikops-new
git pull origin main
./ship.sh
```

**צעד 3 — אם יש merge conflict** על `build.gradle` / `pbxproj`:
```bash
git checkout --theirs frontend/android/app/build.gradle
git checkout --theirs frontend/ios/App/App.xcodeproj/project.pbxproj
git add frontend/android/app/build.gradle frontend/ios/App/App.xcodeproj/project.pbxproj
git rebase --continue
./ship.sh
```

**צעד 4 — Xcode:** Archive → Distribute App → App Store Connect → Upload

**צעד 5 — Android Studio:** Build → Generate Signed Bundle → העלאה ל-Play Console Internal Testing

### בדיקות אחרי Build 5 על הטלפון
- iPhone: swipe מהשוליים השמאליים → חוזר מסך אחורה ✓
- Android: כפתור back חומרה → חוזר מסך אחורה (לא יוצא מהאפליקציה) ✓
- ייצוא PDF מ-ExportModal → share sheet נפתח (iOS: Save to Files; Android: share chooser) ✓
- אייקון במסך הבית של Android → B ממורכז, לא חתוך ✓

### אם האייקון לא נראה טוב על הטלפון
רפליט עשו B ב-58% מהקנבס (Google Safe Zone Standard).
אם לא נראה טוב → ספק חדש: "תעשה B ב-70% במקום 58%" (יותר גדול ויזואלית, אבל עלול להיחתך במכשירים מסוימים).

### Deploy workflow
**שני מצבים לפי CLAUDE.md:**

**1. שינוי JS-only (דפים, טקסטים, באגים):**

## 10. בעיות ידועות / Tech Debt

### ship.sh version drift
ship.sh לא מקמיט versionCode חזרה ל-git. גורם לpull conflicts.
**פתרון לעתיד (משימה #35):**
1. `cat ship.sh | grep versionCode` — לבדוק איפה הלוגיקה (כבר בדקנו: אין)
2. מסקנה: אין bump אוטומטי ב-ship.sh. ה-drift נוצר כש-Cowork או מישהו עורך ידנית.
3. **כלל:** כל version bump עובר דרך רפליט בקומיט נפרד.

### Repo vs. Mac drift
- repo versionCode: 8 (לפני Wave 5) → 10 (אחרי Wave 5)
- Mac versionCode: 9 (uncommitted, מ-Cowork edit בסשן קודם)
- אחרי `git pull origin main` — Replit's 10 ינצח

### Icon concern
האייקון החדש של Wave 5 — B ב-58% מהקנבס. זה Google Safe Zone Standard אבל נראה קטן לעין.
**החלטה:** לראות על מכשיר אמיתי אחרי Build 5. אם באמת לא טוב — ספק תיקון חדש.
