    async def get_daily_weather(city_code, date) -> dict | None
        1. guard: city_code in list, date well-formed, else None
        2. CACHE-FIRST: db.weather_cache.find_one({city_code, date}) → hit =
           return cached value (ZERO HTTP)
        3. fetch isr_cities.xml — house HTTP client, total timeout ≤ 2.5s,
           ONE attempt, no retries; cache-busting query param (?t=epoch) per
           the locked decision (CDN max-age=14d)
        4. parse; requested date must be present in the response, else None
           (covers the early-morning hole and all retro dates)
        5. build {"desc": "תחזית: <WEATHER_CODE_HE[code]>", "temp_min": int,
                  "temp_max": int, "rain_mm": None, "source": "derived"}
        6. upsert into weather_cache {city_code, date, value, fetched_at}
        7. ANY exception anywhere → return None. NEVER raises. No user-facing
           errors ever.
    Index note: weather_cache gets a unique compound index (city_code, date) —
        created in the existing index-bootstrap path.
