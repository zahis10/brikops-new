(units × stages) that PMs / execution engineers will use to track

    routers / project_memberships / billing.

  - Reads qc_template.stages live (computed at render time —
    no migration when admin edits the template).

DIVERGENCES FROM SPEC (also documented at top of router.py)
================================================================
D1. Imports from contractor_ops.router (NOT via qc_router).
    Cleaner dependency graph — qc_router (2,936 LOC) is a consumer
    of these helpers, not their source.

D2. _get_matrix_approver_user_ids includes "active": True filter.
    Spec placeholder omitted it; canonical pattern at qc_router
    L80/L1153/L1853 always filters by active. Inactive (revoked)
    approvers must NOT get matrix-edit access.

D3. "viewer" role removed from _check_matrix_view.
    No "viewer" role exists in the codebase. Aligned to existing
    QC_VIEW_ROLES pattern: super_admin + project_manager + owner +
    management_team. Contractor blocked explicitly.

D4. Stage-scoped approvers (mode="stages") get FULL matrix-edit access.
    Per Zahi 2026-05-04: trust given for any stage approval = trust
    given for matrix annotation. Matrix is project-level, so
    stage-scope on the approver doc is irrelevant.
