================================================================================
PROP INTERFACE:
  {
    open: boolean,
    onClose: () => void,
    allDisciplines: Array<{key, label}>,
    getDisciplineLabel: (key) => string,
    defaultDiscipline: string,
    onDefaultDisciplineChange: (val) => void,
    showFloorField: boolean,
    floorsList: Array<{id, name, ...}>,
    planTypes: Array<{value, label}>  // default has standard +
                                      // tenant_changes
    uploadFn: async (file, discipline, opts) => any,
    onUploadComplete: () => void,
    maxFiles: number = 20,
    maxFileSizeMb: number = 50,
    acceptedExtensions: string = '.pdf,.jpg,.jpeg,.png,.dwg,.dxf'
  }

INTERNAL STATE:
  - bulkFiles, bulkUploading, bulkResults — exact same shape as
    pre-existing ProjectPlansPage state to keep behavior 1:1.
  - bulkFileRef — for clearing the <input>.
  - useEffect on `open` flips false→true → resets all internal
    state. Matches existing closeBulkModal semantics.

HANDLERS (verbatim ported from ProjectPlansPage L461-553):
  - handleBulkFileSelect — appends with default discipline;
    validates count + size.
  - removeBulkFile, updateBulkFile.
  - handleBulkUpload — concurrency=3 loop; calls `uploadFn`.
  - retryBulkFile — single-file retry.
  - closeBulkModal — guarded by `bulkUploading`; calls onClose.
  - handleDefaultDisciplineChange — bubbles via
    onDefaultDisciplineChange + applies to existing pending rows.

JSX: identical structure to original (header + body + per-file row
+ footer + results screen). Backdrop click guarded by
`bulkUploading` (matches original behavior). `dir="rtl"`
preserved. Floor field conditionally rendered via
`showFloorField`. PlanType dropdown options come from `planTypes`
prop.
