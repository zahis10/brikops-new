ARCHITECT REVIEW
--------------------------------------------------------------------------------
Round 1 flagged one real security gap: services/object_storage.generate_url()
returns the RAW stored ref (s3://…) if S3 presigning throws, so a presign
failure could have leaked the key via /auth/me or the upload response.
FIXED: hard guard in _profile_photo_display rejects s3://-shaped results
(→ None fail-soft); upload response routed through the same helper; 3 new
probe checks force the fallback path and assert no leak (probe 22/22).
generate_url() itself was NOT changed — it is shared with worker photos and
task attachments; hardening it globally is out of batch scope (candidate
follow-up).
