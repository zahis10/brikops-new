מה נעשה
-------
נבנה ה-UI המלא לסיורי הבטיחות שה-backend שלהם נבנה ב-4a. משתמש בעל הרשאה
(project_manager / management_team) יכול ליצור סיור, לצעוד בצ'קליסט ולסמן כל
פריט תקין/נכשל/לא-רלוונטי. פריט שנכשל פותח פאנל inline (חומרה חובה, הערה,
תמונות, כותרת ליקוי) — השמירה פותחת ליקוי מקושר בצד השרת. בסוף אפשר להגיש
לחתימה (נעילה) ולפתוח מחדש. החתימות עצמן = batch 4c.

update_tour_item (מונע 200 + audit "item_added" שקרי). זה השינוי היחיד ב-backend.

STEP 5 — SafetyTourRunner.js (חדש — הליבה)
------------------------------------------
- ראש: כותרת (custom_name/label), תאריך, badge סטטוס, "נענו X מתוך Y", כפתור סגירה.
- כל פריט: label + pill קטגוריה + 3 כפתורים תקין/נכשל/לא-רלוונטי (min-h 44px).
- תוצאות SERVER-AUTHORITATIVE: אין flip אופטימי מקומי — pass/na שולחים PATCH
  {result}; בכישלון השורה פשוט נשארת על ערך השרת (אין "בחירה תקועה"). pendingItemId
  חוסם את השורה ומראה spinner בזמן קריאה.
- נכשל = פאנל inline (לא דיאלוג מקונן): chips חומרה (חובה — מראה מקדים ל-422),
  הערה, העלאת תמונות (compressImage→upload→stored_ref, תצוגה דרך photo_display_urls),
  כותרת ליקוי (placeholder = "סיור בטיחות: <label>"). שמירה שולחת
  {result:'fail', severity, note?, photo_urls, defect_title?}.
- re-init effect keyed על [open, tour?.id] בלבד — כדי שה-echo של onChanged
  (runner→parent→prop) לא ימחק state מקומי (פאנל פתוח / טופס) בכל שמירה.
- הוספת פריט ad-hoc (label 2-200 + קטגוריה) בתוך draft.
- footer לפי סטטוס: draft → "הגש לחתימה" (חסום עד שכל הפריטים נענו, עם
  "נותרו X פריטים ללא מענה") + דיאלוג אישור (Dialog נפרד, לא window.confirm) ;
  pending_signature → הודעה + "פתח מחדש לעריכה" ; signed → "הסיור חתום".
- readOnly = t.status !== 'draft' || !isWriter — חוסם כל עריכה/כפתורים.

STEP 6 — SafetyHomePage.js (wiring)
-----------------------------------
- imports: ClipboardList (lucide), 2 הקומפוננטות החדשות, TOUR_TYPE_HE/TOUR_STATUS_HE.
- VALID_TABS: נוסף 'tours' (אחרי 'observations').

- Promise.all הראשי: נוסף listTours({limit:100}) fail-soft (__err) + setTours.
- aux filter effect: early-return גם ל-'tours' (אין סינון צד-לקוח).
- gate כפתור הסינון: מוסתר גם ב-'tours'.
- TabsTrigger "סיורים (N)" + TabsContent → ToursList (רשימה inline: כרטיס לכל
  סיור עם כותרת/תאריך/נענו X/Y/badge סטטוס + כפתור "סיור חדש" ל-writer).
- add-chooser: כפתור 7 "סיור" (ClipboardList) → פותח את דיאלוג היצירה.
- mount: SafetyTourCreateDialog (onCreated → reloadTours + קפיצה ישירה ל-runner)
  + SafetyTourRunner (onChanged/onClose → reloadTours). reloadTours() נוסף.
