WHAT & WHY
----------
The handover signature drawing canvas was unusable: drawing with a
finger or mouse made the whole Vaul drawer sheet "dance" along with
the pointer, so a clean signature was impossible.

Root cause: the <canvas> sits inside a Vaul `Drawer`
(frontend/src/components/ui/drawer.jsx → `import { Drawer } from
"vaul"`). Vaul's drag-to-dismiss gesture catches the pointer motion
in parallel with the canvas draw handlers. `touchAction:'none'` stops
the browser's native touch-scroll but does NOT stop Vaul's JS drag.

Verified in installed Vaul 1.1.2 source:
    frontend/node_modules/vaul/dist/index.mjs:981
    if (element.hasAttribute('data-vaul-no-drag') ||
        element.closest('[data-vaul-no-drag]')) { ... }
→ any element carrying `data-vaul-no-drag` is excluded from
initiating a drawer drag. The documented Vaul escape hatch.

Fix = add the `data-vaul-no-drag` attribute to the signature
<canvas>. One attribute, no logic change. Covers BOTH signature
flows (role/tenant via SignatureSection + legal-section via
HandoverLegalSections) — both render the same SignaturePadModal.
