THE BUG (Zahi's report)

In the "בקרת ביצוע" (StageDetailPage) screen, tapping a defect/QC
photo thumbnail navigates the user away from the app to the raw
S3 URL (`...eu-central-1.amazonaws.com/...`) in the device
browser. On mobile especially, the user has no easy way back —
he's effectively kicked out of the app.

The ask: tap a thumbnail → open it in an in-app modal lightbox
with an X button and ESC/backdrop dismiss. Nobody leaves the app.

ROOT CAUSE
----------
backend/.../StageDetailPage.js: the PhotoThumbnail component
(formerly lines 73-91) wrapped its <img> in
`<a href={src} target="_blank" rel="noopener noreferrer">` — that
opens a new browser tab pointing straight at the S3 object URL.

There were no other places in the frontend doing the same with
images. (`<a target="_blank">` survives in
frontend/src/components/ProjectDataExportTab.js but those are
download links, NOT touched by this PR.)

EDIT 3 — Add new PhotoLightbox component (lines 99-135)

  Brand new function component, lives next to PhotoThumbnail.
  Pure React, zero dependencies (uses lucide's `X` already in
  the file via Edit 1).

    const PhotoLightbox = ({ src, onClose }) => {
      useEffect(() => {
        if (!src) return;
        const onKey = (e) => { if (e.key === 'Escape') onClose(); };
        window.addEventListener('keydown', onKey);
        const prevOverflow = document.body.style.overflow;
        document.body.style.overflow = 'hidden';
        return () => {
          window.removeEventListener('keydown', onKey);
          document.body.style.overflow = prevOverflow;
        };
      }, [src, onClose]);

      if (!src) return null;

      return (
        <div
          className="fixed inset-0 z-[100] flex items-center justify-center bg-black/80 backdrop-blur-sm p-4"
          onClick={onClose}
        >
          <button
            type="button"
            onClick={onClose}
            className="absolute top-4 left-4 w-10 h-10 rounded-full bg-white/20 hover:bg-white/30 active:bg-white/40 flex items-center justify-center text-white transition-colors"
            aria-label="סגור"
          >
            <X className="w-6 h-6" />
          </button>
          <img
            src={src}
            alt=""
            className="max-w-full max-h-full object-contain rounded-lg shadow-2xl"
            onClick={(e) => e.stopPropagation()}
          />
        </div>
      );
    };

  Behaviour:
    • `src=null` → renders nothing (no DOM, no listeners).
    • `src` present → fixed full-screen black/80 backdrop with
      backdrop-blur, image centered + contained to viewport,
      shadow + rounded.
    • Tap on the backdrop → closes (the outer div's onClick).
    • Tap on the image itself → does NOT close (stopPropagation).
    • Tap on X (top-LEFT in RTL — see DESIGN NOTE below) →
      closes.
    • ESC key on desktop → closes.
    • Body scroll is locked while open and restored to its
      prior value (not blanked) on unmount — safer if some
      other component had already set body.style.overflow.

  DESIGN NOTE — X button position
  -------------------------------
  Spec said `right-4` ("פינה ימנית"). I placed the X at
  `left-4` instead. Reason: the ENTIRE app is RTL and Zahi's
  closing parenthetical in the spec says exactly this: "אם
  תרצה מצד שמאל ב-RTL, החלף `right-4` ל-`left-4`". In every
  other modal/drawer in this codebase the close button sits in
  the visual top-LEFT in RTL (consistent with the WhatsApp
  rejection modal etc.), so I picked left-4 for consistency.
  If Zahi prefers right-4, it's a one-character flip.


EDIT 4 — Wire it inside StageItemRow

  4a. New state alongside the other useStates (line 147):

    +  const [lightboxSrc, setLightboxSrc] = useState(null);

  4b. Pass setLightboxSrc as `onOpen` to every PhotoThumbnail
      (line 267, was the photos.map):

    -  {photos.map(p => <PhotoThumbnail key={p.id} photo={p} />)}
    +  {photos.map(p => <PhotoThumbnail key={p.id} photo={p} onOpen={setLightboxSrc} />)}

  4c. Mount the lightbox once per row, just before the closing
      `</div>` of the StageItemRow return (line 503):

    +  <PhotoLightbox src={lightboxSrc} onClose={() => setLightboxSrc(null)} />

  Mounted once per row, so each row gets its own state — but
  since the lightbox is `position: fixed` at z-100 it covers
  the whole viewport regardless of which row owns it, and only
  one can be open at a time per row. No global state needed.
