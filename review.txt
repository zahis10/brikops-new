WHAT & WHY
--------------------------------------------------------------------------------
First offline-WRITE batch. Today, marking a QC item offline fails: handleSave
PATCHes each changed item via qcService.updateItem, and offline those reject →
"שגיאה בשמירה" and the worker's marks are LOST.

This batch lets a worker mark QC items (status תקין/לא-תקין + note) OFFLINE,
keeps the marks on screen, persists them across an app restart, and auto-syncs
on reconnect. Everything gated on FEATURES.OFFLINE_MODE (currently true). Flag
OFF = today's behavior exactly (no outbox DB, online save path byte-for-byte).

Scope (narrow): QC status + note ONLY. NO photos (BATCH 3b). Approve/submit/
reject stay ONLINE-gated (never queued). Conflict: last-write-wins on replay.

PER-FILE CHANGES (2 NEW + 2 EDITS)
--------------------------------------------------------------------------------
NEW  frontend/src/services/offlineOutbox.js
  Dependency-free IndexedDB write queue. DB "brikops-outbox", store
  "qc_item_updates", keyPath "key" = `${runId}::${itemId}` (re-marking an item
  OVERWRITES its op → one op/item, local last-write-wins). Fail-soft (never
  throws; missing/blocked IndexedDB → null/[]/0/no-op). Records {key,runId,
  itemId,payload,ts,failed}. No PII logging (notes never logged). Exports:
    enqueueQcUpdate(runId,itemId,{status?,note?})
    getQcUpdatesForRun(runId) → { [itemId]: {status?,note?} }
    getAllQcUpdates() → [{key,runId,itemId,payload,ts,failed}]
    removeQcUpdate(runId,itemId), markQcUpdateFailed(runId,itemId), outboxCount()

NEW  frontend/src/services/offlineSync.js
  Render-less flush-on-reconnect engine. Imports qcService from ./api (no raw
  axios). flushOutbox() → {synced,failed}:
    - no-op if !FEATURES.OFFLINE_MODE or navigator.onLine===false
    - re-entrancy guard (_flushing); throttle 3 concurrent; never throws
    - per op await qcService.updateItem(runId,itemId,payload):
        * 2xx RESOLVED  → removeQcUpdate   ← THE ONLY removeQcUpdate call site
        * !error.response (NETWORK) → KEEP (retried next flush)
        * error.response (4xx/5xx) → markQcUpdateFailed + KEEP (never dropped)
    - if synced>0 → dispatch CustomEvent('brikops:outbox-synced',{detail:{runIds,
      failed}})
  startOutboxSync() — idempotent (_started guard): addEventListener('online',
  flush) + visibilitychange→(visible&&online)→flush + one startup flush.
