מה נעשה (סקירה קצרה)
--------------------
הופרד סוג רשומה חדש "תיעוד" (observation) מתוך רשומות הבטיחות הקיימות
(defect), באמצעות שדה מבחין (discriminator) בשם `kind` על אוסף
safety_document. תיעוד = רשומה ללא חומרה/סטטוס/אחראי, לתצוגה בלבד; ליקוי =
כפי שהיה. רשומות ישנות ללא `kind` נשארות "ליקוי" לכל דבר.

BACKEND
-------
- schemas.py: SafetyDocumentKind enum + שדה kind ב-SafetyDocument; severity
  הפך ל-Optional.
- safety_router.py:
  * SafetyDocumentCreate: kind + severity Optional + model_validator
    (ליקוי מחייב severity / תיעוד אוסר severity+status) עם שגיאות בעברית.
  * create_document: כותב kind ו-severity מותנה.
  * update_document: guard לתיעוד — 422 אם נשלח severity/status/resolved_at.
  * list_documents: param kind — defect => kind != observation (כולל legacy),
    observation => kind == observation.
  * create_task: חסימת משימה מתקנת כנגד ref שהוא תיעוד.
  * docs aggregation + score: מדלגים על תיעוד.
  * Excel/PDF: פיצול ליקויים/תיעוד + גיליון/בלוק "תיעוד" נפרד.
  * export_safety_filtered: param kind (ברירת מחדל — ללא סינון).
- services/safety_pdf.py: פיצול ליקויים/תיעוד; סעיף 5 על ליקויים + "5ב. תיעוד";
  סעיף 8 סיכומי קטגוריה על ליקויים בלבד + שורת סה"כ תיעוד.
- server.py: backfill אידמפוטנטי ב-startup — kind='defect' לרשומות ללא kind.

FRONTEND
--------
- safetyLabels.js: KIND_HE.
- SafetyHomePage.js: טאב "תיעוד" (obsDocs+filteredObs), fetch שביעי
  kind:'observation', קריאות documents שולחות kind:'defect', סינון עזר
  לתיעוד (category+date_from+date_to), reload מרענן שתי הרשימות,
  בורר הוספה עם כפתור "תיעוד", docForm נושא kind. רכיב ObservationsList חדש.
- SafetyDocumentForm.js: prop kind — מסתיר severity/status/assignee, מדלג על
  ולידציית חומרה, משמיט severity לתיעוד, שולח kind ביצירה.
- SafetyDocumentDetail.js: כותרת/Badge ירוק "תיעוד", הסתרת severity+status.
- SafetyFilterSheet.js: בלוק סינון תיעוד (קטגוריה+תאריכים) + EMPTY_FILTER_OBSERVATIONS.
- SafetyExportMenu.js: יצוא מסונן מקבע kind:'defect' (ראה תיקון למטה).

ביקורת קוד (architect) — ממצאים וטיפול
--------------------------------------
תוצאה: הפיצול נכון ברוב הנתיבים (CRUD/read/score/bulk-delete/יצוא כללי),
guard ה-422 תואם, ואין ממצאי אבטחה.

ממצא חוסם יחיד שנמצא — וטופל:
* יצוא מסונן (export_safety_filtered) לא קיבל kind מה-UI, כך שיצוא "לפי סינון"
  של ליקויים היה עלול לכלול רשומות תיעוד שתאמו קטגוריה/תאריך.
  תיקון: SafetyExportMenu.js מזריק kind:'defect' לפרמטרי היצוא המסונן (התפריט
  קשור תמיד למסנן הליקויים). כך תיעוד + רשומות legacy לא דולפים ליצוא הליקויים.

המלצות פתוחות (לא חוסמות, לשיקול בהמשך):
- בדיקת אינטגרציה ל-/export/filtered שמאמתת בידוד kind.
- בדיקת UI/מוק שמאמתת שפרמטרי היצוא כוללים kind לפי הטאב הפעיל.
