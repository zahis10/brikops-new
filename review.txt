LOCATION 2 — router.py: NEW _resolve_request_org_id helper

File: backend/contractor_ops/router.py  (L487-527, inserted before

ADDED:
+  # BATCH J (2026-05-11) — Scope-aware paywall helper.
+  # ... (10 lines of comments)
+  _ORG_URL_PATTERNS = [
+      (_re.compile(r'^/api/projects/([^/]+)'),  'projects',  'org_id'),
+      (_re.compile(r'^/api/tasks/([^/]+)'),     'tasks',     'project_id'),
+      (_re.compile(r'^/api/buildings/([^/]+)'), 'buildings', 'project_id'),
+      (_re.compile(r'^/api/floors/([^/]+)'),    'floors',    'project_id'),
+      (_re.compile(r'^/api/units/([^/]+)'),     'units',     'project_id'),
+  ]
+
+  async def _resolve_request_org_id(request_path, db) -> Optional[str]:
+      for pattern, collection, field in _ORG_URL_PATTERNS:
+          m = pattern.match(request_path)
+          if not m: continue
+          entity_id = m.group(1)
+          entity = await db[collection].find_one(
+              {'id': entity_id}, {'_id': 0, field: 1}
+          )
+          if not entity: return None
+          if field == 'org_id':
+              return entity.get('org_id')
+          project_id = entity.get('project_id')
+          if not project_id: return None
+          project = await db.projects.find_one(
+              {'id': project_id}, {'_id': 0, 'org_id': 1}
+          )
+          return project.get('org_id') if project else None
+      return None

Reuses module-level `_re` from L10. Returns None for non-matching
paths so callers fall back to legacy behavior (no regression).

PERFORMANCE NOTES
================================================================================
- Happy path (~95% of traffic, single-org users): UNCHANGED. Single
  DB query for subscription, same as today. Test 1 enforces this
  invariant — `_resolve_request_org_id` literally never called.

entirely now go through correctly (huge UX win, no measurable cost).
