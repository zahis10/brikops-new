## What this ships

1. **Bug B fix** Рђћ floor-shared template items (no `unit_id` field) now sync to ALL units on the floor. The 4 existing hooks (update_qc_item, approve, override, reject_stage) used `[i for i in items_all if i.get("unit_id") == uid]` which filtered to `[]` for floor-shared items Рєњ empty items list Рєњ `_compute` returned None Рєњ no sync. New pattern: `items_have_unit_id = any(i.get("unit_id") for i in items_all)`; if False, pass items_all to every unit's sync call.

2. **Bug C fix** Рђћ `submit_stage` (PM submits to senior) was completely unhooked. Matrix never turned ORANGE "ОъОъОфОЎОЪ ОюОљОЎОЕОЋОе" until approve/reject. Added BackgroundTasks param + sync hook calling helper with `stage_status="pending_review"`.

3. **STOP-GATE 0.6 finding** Рђћ STEP 0 audit found 2 MORE unhooked mutation sites that the spec didn't anticipate. Both ship in this batch:
   - `reject_qc_item` L1478 Рђћ per-item rejection cascades stage to "reopened", was unhooked
   - `reopen_stage` L2980 Рђћ manual senior reopen, was unhooked
   - Both call sync with `stage_status="reopened"` which falls through in `_compute_matrix_status_from_qc` to item-level activity (in_progress if any item touched, else None). NO mapping logic change Рђћ only a docstring + inline comment.

4. **Feature D Рђћ `ready_for_work` status** Рђћ new MANUAL-only planning state ("­ЪЊд ОъОЋОЏОЪ ОюОбОЉОЋОЊОћ", cyan-100, PackageCheck icon). PMs select it from CellEditDialog. Per D3, QC sync ALWAYS WINS Рђћ when QC starts tracking the same stage, sync overwrites manual cell.

5. **D6** Рђћ `MatrixCellUpdate.status` Pydantic Literal at schemas.py L953 += `'ready_for_work'`. **Intentionally OMITS** `'pending_review'` Рђћ sync-only statuses must never be settable via manual PATCH. Test T19 enforces this with `pytest.raises`.
