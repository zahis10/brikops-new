DESIGN DECISIONS (per approval)
-------------------------------
D1a — Backend is the right layer.
    The earlier scoping note assumed taskService.list could power a client-side
    safety filter, but BuildingDefectsPage doesn't fetch tasks — it consumes
    the defects-summary aggregate. Rather than add a second round-trip, we
    extended the aggregate with safety_open_count, mirroring the additive
    categories[] field at L1489. Single fetch, instant toggle, no API churn.

D2a — Inline toggle, not FilterDrawer extension.
    FilterDrawer is shared across BuildingDefectsPage, ProjectControlPage,
    UnitDefectsPage and uses a sections-only API (multi-select arrays). A
    boolean toggle would have required either a new section type or a
    bool-as-array hack. Inline above the drawer trigger keeps the shared
    component clean, matches the discovery-first chip pattern from
    ProjectControlPage L3849, and gives the toggle a ≥44px touch target.
