WHAT SHIPPED — 5 BUGS FIXED
---------------------------
Bug A (CRITICAL — flood):
  Cron now iterates by `(project, assignee_user)` instead of
  `(project, company)`. A contractor with tasks in the same project
  tagged under N companies receives ONE digest with count=N, not N
  separate "1 ליקוי" messages. Cooldown re-keyed to (project, user)
  via new `_check_cooldown_for_user`.

Bug B (routing):
  `App.js ProjectsHome` inverted. Defines
  `PM_OR_ADMIN_ROLES = ['project_manager','owner','management_team']`
  and treats `platform_role === 'super_admin'` as PM/admin too.
  Everyone else (contractor + sub-roles like execution_engineer,
  site_manager, future field-roles) → ContractorDashboard.

Bug C (inactive projects):
  `send_all_contractor_reminders` filters projects to
  `status='active' AND ($or archived missing/false/null)`.
  Suspended/archived/deleted projects no longer trigger sends.

Bug D (dormant users >45d):
  New `_is_user_dormant(user)` helper. Primary signal is
  `last_login_at`; falls back to `created_at` so brand-new invites
  (no login yet, account <45d old) still receive their first
  reminder (V2 new-user exception per Replit feedback). Both
  missing/malformed → defensive dormant.

Bug E (WA toggle):
  New `_user_disabled_whatsapp(user)` helper. Returns True iff
  `whatsapp_notifications_enabled is False` (default-True opt-in
  preserved). Wired into the 3-tier eligibility gate of
  `send_contractor_reminder` BEFORE dormancy/prefs/phone checks.

Orphan fallback (future-proof — current prod has 100% assignee_id):
  `send_contractor_reminder_to_company` handles tasks with
  `company_id` but no `assignee_id`. One Hebrew digest per
  (project, orphan_company), separate cooldown via
  `_check_cooldown_for_company`. Skips dormant + WA-toggle gates
  (no user to evaluate). Stream A v2 `send_pm_digest` UNTOUCHED.
