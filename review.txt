STEP 0 FINDINGS (verified before coding)
--------------------------------------------------------------------------------
(a) gate_status worker-branch end: in gate_public.py the worker lookup is
    `tok = await db.worker_entry_tokens.find_one({"token": token, ...})`;
    when tok is None the old code returned the neutral {"state":"invalid"}.
    The guest branch slots in exactly there — inside `if not tok:` we first
    try guest_entry_passes by token; only if THAT also misses (or the pass
    is revoked) do we return the same neutral {"state":"invalid"}. The
    worker green/red path below is untouched.
(b) validate_upload WITHOUT a user — ANSWER: contractor_ops' validate_upload
    keys its rate-limit/quota bookkeeping on the authenticated user id, so it
    CANNOT be used on the public guest-sign endpoint (there is no user).
    Per the spec's fallback, the public endpoint does INLINE validation
    instead: hard 600KB byte cap via read(cap+1) length check, PNG magic
    bytes (\x89PNG\r\n\x1a\n) AND content-type image/png → otherwise 422.
    The per-IP _check_throttle (same helper the worker gate uses) runs first.
(c) "הנפק קוד אורח" button home: SafetyHomePage's workers tab header renders
    the "הוסף עובד" action for isWriter users; the amber guest button was
    placed alongside it, and GuestPassSection renders inside the workers
    TabsContent below GateScansSection.

SECURITY (the new public WRITE surface — all server-side)
--------------------------------------------------------------------------------
- Writes possible ONLY for a live, active, UNSIGNED pass found by token;
  unknown/revoked/worker tokens → neutral 404, nothing written.
- Sign-once lock is atomic (re-asserted in the update filter) — concurrent
  double-sign cannot produce two writes; loser gets 409 "התדריך כבר נחתם".
- 600KB hard cap read BEFORE full buffering (read cap+1), PNG magic bytes +
  content-type both required, signer_name capped at 120 chars (architect
  hardening suggestion, implemented).
- Per-IP _check_throttle applied (same as worker gate).
- No PII beyond first name+company on any public response; signature_ref
  never leaves the server on public routes (probe asserts this).
- gate_public.py still imports NO authenticated dependencies.
