=== Task #109 — Company Contact Fallback in Contractor Dropdown ===

========================================
INVESTIGATION: Backend notification recipient resolution
========================================

notification_service.py _resolve_target_phone (lines 323-350):
  1. First checks assignee_id → looks up user → gets phone_e164
  2. If no assignee, falls back to company_id → looks up companies collection → gets phone_e164

BUT: The fallback checks self.db.companies (global), not project_companies.
Project companies store contact_phone/contact_phone_raw, not phone_e164/phone.
So the company phone fallback does NOT work for project companies.

DECISION: Rule #1 applied — UI fallback only, no WhatsApp from contact.
  - Selecting company contact sets company_id, keeps assignee_id null
  - No fake user IDs ever stored in DB
