WHAT SHIPPED
------------
Two new bell-notification events on the defect lifecycle:

  Event 1 — defect_close_request:
    When a contractor uploads proof and requests defect closure
    (status → pending_manager_approval), every project_manager +
    management_team member of that project gets a bell notification:
    "{contractor} ביקש לסגור ליקוי \"{title}\" — ממתין לאישורך"

  Event 2 — defect_status_change_by_pm:
    When a PM approves or rejects the contractor's proof, the
    contractor (task assignee) gets a bell notification:
    approve → "המנהל אישר את התיקון של \"{title}\""
    reject  → "המנהל דחה את התיקון של \"{title}\" — {reason}"

Existing QC-stage notifications on qc_notifications collection are
untouched (legacy collection name kept per Zahi 2026-05-01 — no DB
migration). New docs add a `notification_type` field; existing QC
docs without it are treated as type 'qc' by the list endpoint.
