================================================================================
INTENTIONAL BEHAVIORS (no code action; documented for Zahi)
================================================================================
- from=dashboard SURVIVES tab switches inside /control. setActiveTab uses
  setSearchParams(prev => {...}, {replace:true}) at line 3417 and merges
  existing params. So a user who arrives via /control?...&from=dashboard,
  switches to the "מבנה" tab, then taps the back arrow → still returns to
  /dashboard. Intentional: dashboard context follows the user during their
  session inside /control.
- Bookmark edge case: if a user bookmarks a URL with from=dashboard and
  opens it next session, the back arrow will go to /dashboard even though
  the visit technically wasn't "from dashboard". Acceptable tradeoff —
  bookmarks of deep-linked URLs with query params are rare, and the worst
  case is back goes to /dashboard instead of /projects (still sensible).
- In-page לאישור shortcut at ProjectControlPage:3770 — clicking it from a
  /control session that arrived with from=dashboard DROPS the from param
  (because navigate(`<full URL>`) replaces all search params, unlike the
  setSearchParams merge used for tab switching). After clicking, back arrow
  reverts to /projects. Minor edge case, flagged in audit, out of scope.
