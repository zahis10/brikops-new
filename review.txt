STEP 0 — FINDINGS / DESIGN DECISIONS
------------------------------------
1. Token model: worker_entry_tokens collection. token = secrets.token_urlsafe(24)
   (32 chars, ~192 bits entropy). Fields: id, token, org_id, project_id,
   worker_id, status(active/revoked), created_at, revoked_at, rotated_from, qr_ref.
2. Single-active-token invariant is enforced at DB level: partial unique index
   uidx_wet_active on (project_id, worker_id, status) where status=="active".
   ensure_entry_token / rotate_entry_token handle DuplicateKeyError atomically
   (concurrent ensure returns existing token; concurrent rotate converges
   last-writer-wins to exactly one active token). Added after architect review
   flagged the read-then-write race; verified by concurrency probes V9a/V9b.
3. Public status endpoint /api/gate/{token} (gate_public.py):
   - Separate router, NO auth import; registered in server.py only under
     ENABLE_SAFETY_MODULE.
   - Neutral {"state":"invalid"} for unknown/revoked token or missing worker —
     no information leak.
   - Decision matrix: red if worker blocked or induction missing/expired;
     green otherwise, with warnings array (expiry-soon etc.).
   - X-Robots-Tag: noindex, nofollow on every response.
   - Throttling is two-layer: the app-wide unauthenticated per-IP limiter
     (30 req/min, Mongo-backed, in server.py middleware) fires first; the
     module's own in-memory 60/min per-IP throttle is defense-in-depth.
   - Every scan (valid or not) written to gate_scan_log.
4. Writer endpoints (gate.py), restricted to SAFETY_WRITERS
   (project_manager, management_team):
   - GET  .../workers/{id}/entry-token       (ensure-on-demand)
   - POST .../workers/{id}/entry-token/rotate
   - GET  .../workers/{id}/entry-qr.png      (QR PNG, qrcode lib, stored at
     safety/{project_id}/entry-qr/{token_id}.png via storage backend)
   - PATCH .../workers/{id}/block            (+ audit event worker_block_change)
   - GET  .../gate-scans                     (scan-log list)
   Contractor gets 403 on block/rotate (probe V4e).
5. Worker lifecycle hooks (workers.py): on create — ensure token + optional WA
   send; on delete — revoke all tokens (best-effort).
6. WhatsApp auto-send: behind WA_ENTRY_QR_ENABLED (default FALSE — zero sends
   unless explicitly enabled). Template name WA_TEMPLATE_WORKER_ENTRY_QR.
   _send_wa_template extended with header_image_url=None keyword (approved
   exemption); header component added only for https URLs; all existing call
   sites unaffected (byte-identical behavior, probe V6f).
7. Frontend:
   - /gate/:token public page (GatePage.js): mobile-first RTL, green/red/
     invalid states + warnings, no auth header on the public API call.
   - SafetyWorkerCard.js: "קוד כניסה" section — QR download/share/print,
     rotate (with confirm), block toggle + חסום chip.
   - SafetyHomePage.js: gate-scans section under תיעוד tab (writers only),
     חסום chip in workers list.
8. New indexes: uidx_wet_token (unique token), idx_wet_project_worker,
   uidx_wet_active (partial unique), idx_gsl_project_ts — 34 safety indexes total.
9. ios/ and android/ untouched. requirements.txt: qrcode==8.2 pinned.

V9a (10 parallel ensures → 1 active), V9b (10 parallel rotates → 1 active),
V9c (index present & unique). No other serious authz/injection/leak findings.
