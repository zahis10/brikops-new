    fix(auth): auto-logout on mid-session 401 + redirect to /login
    
    When a user's JWT expires mid-session, every subsequent API call
    returns 401 with no global handler. The UI shows a cascade of
    "לא נמצא" / "שגיאה בטעינה" errors and users don't realize they're
    logged out.
    
    Fix: a second response interceptor inside AuthProvider catches 401
    on non-auth endpoints while a token exists. Clears token + cookie +
    user state, shows "החיבור פג, אנא התחבר מחדש" toast, redirects to
    /login via full page reload (AuthProvider is outside BrowserRouter
    so useNavigate isn't available — reload also guarantees clean
    post-session state).
    
    Guards: skips /auth/ URLs (failed login/OTP legitimately 401),
    skips when no token, debounced via ref so concurrent 401s fire the
    flow exactly once. Respects #171 (no 401 handling in api.js).
    Existing pending_deletion 403 handler, fetchCurrentUser, and
    _isAuthError untouched.
    
    Deviation: spec wrapped the logic in try/catch with console.warn
    and a 17-line rationale comment. Code review flagged both as
    inconsistent with the neighboring pending_deletion interceptor's
    style. Refactored to match — same logic, same guards, same redirect.
    
    One file changed (frontend/src/contexts/AuthContext.js, +24 lines).
    No deps, no backend changes.
