  Рђб Added 4 small refs:
      - `latestLoadMoreRef` / `latestIsContractorRef`: kept in sync
        with current values via tiny mirror effects (lines 290-291).
        This avoids stale closures when filters change between
        navigations.
      - `lastRefreshAtRef`: 500ms guard so the focus + visibility
        events that both fire on tab return don't double-fetch.
      - `didMountRef`: skips the very first run so we don't double-
        fetch with the existing init effect (L230-273) on first mount.
  Рђб New useEffect (lines 293-321) that:
      - Triggers on `location.key` change (SPA back/forward) and
        `project?.id` change Рєњ calls `refresh()` from the second run
        onward (skipped on initial mount via `didMountRef`).
      - Listens to `window focus` and `document visibilitychange`
        events (return-from-tab and return-from-mobile-background).
      - Refresh fn: clears `taskIdsRef`, resets offset to 0,
        sets hasMore=true, then calls
        `latestLoadMoreRef.current(0, latestIsContractorRef.current)`
        so the call always uses the freshest filter closure.
      - Cleans up listeners on unmount.

WHY THIS IS SAFE (no infinite loop, no double-fetch):
  Рђб Initial mount: `didMountRef.current === false` Рєњ only listeners
    are registered. The init effect at L230-273 does the first load.
  Рђб Filter change: init effect fires (good). My new effect's deps
    don't include filters, so it doesn't re-run. The `loadMore`
    closure used by the listeners is always the latest via ref.
  Рђб Back-navigation in SPA: `location.key` changes Рєњ my effect runs,
    `didMountRef` is now true, refresh() fires. Init effect deps
    unchanged Рєњ it doesn't fire. Single fetch.
  Рђб Tab return: focus + visibilitychange both fire within ~ms; the
    500ms `lastRefreshAtRef` guard collapses them into one fetch.
