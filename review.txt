BATCH I — Building defects contractor leak (P0 SECURITY hotfix)
================================================================================
Date:           2026-05-11
Branch:         staging  (NOT main)
Severity:       P0 — DATA LEAK / privacy violation.
                Two-bug compound (caught by Zahi smoke during BATCH G):
                1. Frontend (TaskDetailPסשage back nav): contractor entering
                   via WhatsApp deep-link routes back to
                   `/projects/{}/buildings/{}/defects` (BuildingDefectsPage).
                2. Backend (`/buildings/{id}/defects-summary`): allows ANY
                   project member, including contractors, via the
                   permissive `_check_project_read_access` helper.
                Combined: contractor sees all building defects across
                companies — reportable under Israeli privacy law if
                shipped to prod.

Approach:       Defense-in-depth (per Zahi 2026-05-11):
                - Backend: 403 for non-management roles (canonical guard).
                - Frontend: contractors never navigate to building defects
                  page (deep-link safety branch + don't pre-set
                  sessionStorage to it).
                Either layer alone would prevent the leak; both together
                protect against URL hand-crafting, bookmarks, cached FE,
                future regressions, undiscovered entry points.
