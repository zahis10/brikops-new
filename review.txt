KEY IMPLEMENTATION DECISIONS
--------------------------------------------------------------------------------
• Uniqueness (decision 1): Mongo partialFilterExpression CANNOT express
  {deletedAt: null} → unique COMPOUND index (project_id, diary_date, deletedAt)
  instead. With no delete endpoint in D1 every doc has deletedAt=None, so
  active-per-day uniqueness is exact. Friendly Hebrew 409 via pre-check;
  DuplicateKeyError caught → same 409 (index closes the race window).
• Derivation (decisions 6+9): 7 sections derived at create; EVERY source wrapped
  in its own try/except → failure yields an empty section, never a 500. Zero
  coupling to ENABLE_SAFETY_MODULE — empty collections simply derive empty.
  Every derived item carries source:"derived"; manual items carry "manual".
• defect_counts (documented heuristic per batch): opened = defects created that
  date; closed = same-day updated_at with status ∈ resolved/verified. No audit
  scan.
• entered_late (decision 4): diary_date < israel_today(), server-computed at
  create, absent from the PATCH model → immutable.
• Signature (decision 2): mirrors sign_training 3c chain EXACTLY — rate-limit →
  validate_upload → read → empty-check → check_upload_bytes →
  check_storage_quota(org) → save_bytes(diaries/{pid}/{eid}/sig_{uuid}.png) →
  record_upload; typed variant; Hebrew 422s. ATOMIC claim: update_one filter
  {status:"draft", worker_signature:None} → modified==0 → 409. Key-only at
  rest; signature_display_url regenerated per-GET, never persisted.
• Signed immutability (decision 5): PATCH + refresh-derived ALSO re-assert
  status:"draft" INSIDE their update_one filter (matched==0 → 409) — this
  closes the read→write race vs a concurrent signer. Added after architect
  review flagged the window; race probes added (V5b).
• second_signature: reserved field, stored as None, NO setter anywhere (D1).
• SAFETY_WRITERS imported from safety._shared — role-tuple import only; module
  import has no route side effects when the safety flag is off (verified V7).
• Audit entity "work_diary": created / updated / derived_refreshed / signed /
  addendum_added.
