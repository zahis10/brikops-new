WHAT
----
PHASE 3b of upload-abuse-hardening. Wired the 5 helpers
(check_upload_rate_limit, check_content_length, check_upload_bytes,
check_storage_quota, record_upload) into ALL 5 unprotected upload
endpoints in handover_router.py:

  A. PUT  /organizations/{org_id}/logo               (2MB cap)
  B. POST .../meter-photo/{meter_type}               (5MB cap)
  C. POST .../tenants/{tenant_idx}/id-photo          (8MB cap, PII)
  D. PUT  .../signatures/{role}                      (no per-file cap)
  E. PUT  .../legal-sections/{section_id}/sign       (no per-file cap)

WHY
---
P2 security. Before this batch these 5 routes had ONLY their
per-file MAX_SIZE check — no per-user count cap, no per-user
byte-rate cap, no Content-Length fast-reject, no per-org storage
quota, no ledger. A single authed user could spam any of them and
fill the org's S3 bucket. 3a (qc_router) closed the QC route; 3b
closes the handover routes; 3c (safety+excel+import) follows.

ARCHITECTURE NOTES (read before review)
---------------------------------------
1. `Request` was already imported on L1 — added `request: Request`
   only to A/B/C signatures (after path params, before file=File(...)).
   D/E got NO `request` param: spec deliberately omits
   check_content_length on signatures (no per-file cap there), so
   adding an unused param would be noise.

2. B/C — moved `db = get_db()` UP from after the storage call to
   right after `_check_not_locked(protocol)`. The org-lookup
   (`db.projects.find_one(...)`) needs db BEFORE storage, and
   get_db() is idempotent. No double-call.

3. D/E — Use-before-assignment risk inside the `if signature_type
   == "canvas":` block. Required order is enforced:
   check_upload_rate_limit → validate_upload → read image_data →
   <100 check → check_upload_bytes → _proj lookup → _org_id = (...)
   → check_storage_quota → save_bytes → record_upload. The
   `signature_type == "typed"` branch is COMPLETELY UNAFFECTED
   (no file, no hooks).

4. A — ledger uses len(raw), but stored bytes are resized (PIL
   thumbnail ≤400x400). Deliberate ≤2MB over-count, explicit
   tradeoff per plan §step 2.

5. A — Content-Length cap uses literal `2 * 1024 * 1024` (5MB / 8MB
   for B/C) because the function-local `MAX_SIZE` is not yet defined
   at the top of the body.

6. Quota fails OPEN on DB error; record_upload is best-effort.
   Identical semantics to PHASE 2 / PHASE 3a (qc_router).
