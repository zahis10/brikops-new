TWO BUGS FIXED IN THIS BATCH
================================================================================

BUG A — Contractor click on a project in MyProjectsPage no-ops
  Root cause:  navigation.js L10-11 sent contractors to /projects (where
               they already are). For a PM-globally-but-contractor-here
               user, ProjectsHome inspects user.role='project_manager'
               → returns same MyProjectsPage. No-op.
  Fix:         navigateToProject now passes /projects/<id>?src=contractor.
               App.js escape hatch (originally for ?src=wa, Batch 6C)
               extended to also accept ?src=contractor — ProjectsHome
               then forces ContractorDashboard with initialProjectId set.

BUG B — PM cannot assign defect to "company-only" when company HAS contractors
  Root cause:  TaskDetailPage L1092-1094 contactFallbacks filter
               !projectContractors.some(...) excludes companies with any
               registered contractor. After P0 batch (any company member
               can submit proof), this gap left PMs no way to set up
               that flow.
  Fix:         New companyOnlyOptions list emits "🏢 <name> — כל החברה"
               for EVERY project_company. Added before contactFallbacks
               in the InlineSelect spread. Empty-state guard +
               label-resolver updated. Backend handleAssignContractor
               already supports __contact__<id> via L640-644.
