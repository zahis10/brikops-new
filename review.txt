ROOT CAUSE OF v1 FAILURE — DEPS ARRAY TDZ
================================================================================
JavaScript `const` declarations are subject to TDZ — accessing the
binding before its declaration line throws ReferenceError.

In React render, declarations execute TOP-TO-BOTTOM:
  L92-103:  viewport-meta useEffect
  L105-167: imageFile useEffect with doLoad inside
  L172:     getLineWidth = useCallback(..., [])
  L174:     redraw = useCallback(..., [getLineWidth])
  L262:     applyDisplayScale = useCallback(..., [redraw])   ← LATER
  L287:     resize useEffect with deps [loaded, applyDisplayScale]

v1's deps array `[imageFile, applyDisplayScale]` at L156 evaluated
applyDisplayScale BEFORE its declaration at L262 → ReferenceError →
Hebrew error boundary fires → entire app fallback page.

The closure inside doLoad (`async function doLoad() { ...
applyDisplayScale() ... }`) does NOT throw at function creation — it
captures the binding name lexically. Only the deps ARRAY throws
because it's a synchronous expression that reads the binding.

Why the v2 eslint-disable is SAFE:
  1. applyDisplayScale identity is STABLE — its useCallback deps are
     `[redraw]`; redraw deps are `[getLineWidth]`; getLineWidth deps
     are `[]`. So applyDisplayScale never changes identity across
     renders.
  2. The closure inside doLoad always references the latest stable
     applyDisplayScale (no stale-closure issue because identity
     doesn't change).
  3. ESLint exhaustive-deps still enforces the chain on each
     useCallback line. If a future maintainer makes redraw unstable
     (adds an unstable dep), ESLint catches it on the redraw line —
     not on the suppressed line.
  4. By the time React COMMITS and runs the effect (post-render),
     all consts above are initialized → no TDZ at the call site
     inside doLoad.

The 14-line warning comment in LOCATION 3 cites v1's revert and the
TDZ trap explicitly so future maintainers don't "fix" it back.

LOCATION 1 — Extract applyDisplayScale callback
================================================================================
File: frontend/src/components/PhotoAnnotation.js
Inserted: AFTER `redraw` useCallback closing `}, [getLineWidth]);`
          @ pre-edit L243 (post-edit L257-280).

ADDED (L259-280):
  // BATCH F-keyboard (2026-05-12) — recompute canvas display
  // dimensions based on the current visual viewport. Called on
  // image load AND on viewport resize (mobile keyboard pop). The
  // intrinsic canvas resolution (canvas.width / canvas.height)
  // stays fixed — we only update CSS display size + scaleRef.
  // After resize, redraw refreshes the bbox cache for hit-testing.
  const applyDisplayScale = useCallback(() => {
    const canvas = canvasRef.current;
    const img = imgRef.current;
    if (!canvas || !img) return;
    const w = canvas.width;
    const h = canvas.height;
    if (!w || !h) return;
    const vv = window.visualViewport;
    const containerW = vv ? vv.width : window.innerWidth;
    const containerH = (vv ? vv.height : window.innerHeight) - 120;
    const displayScale = Math.min(containerW / w, containerH / h, 1);
    scaleRef.current = displayScale;
    canvas.style.width = Math.round(w * displayScale) + 'px';
    canvas.style.height = Math.round(h * displayScale) + 'px';
    redraw(strokesRef.current);
  }, [redraw]);

LOCATION 4 — visualViewport.resize + window.resize listener useEffect
================================================================================
File: frontend/src/components/PhotoAnnotation.js
Inserted: AFTER applyDisplayScale (LOCATION 1) @ post-edit L282-302.

ADDED (L282-302):
  // BATCH F-keyboard (2026-05-12) — listen for viewport changes
  // (mobile keyboard appearance is the primary use case) and
  // recompute the canvas display scale. Without this, the canvas
  // keeps its original dimensions and gets clipped behind the
  // keyboard, appearing zoomed-in.
  useEffect(() => {
    if (!loaded) return;

    const vv = window.visualViewport;
    if (vv) {
      vv.addEventListener('resize', applyDisplayScale);
    }
    window.addEventListener('resize', applyDisplayScale);

    return () => {
      if (vv) {
        vv.removeEventListener('resize', applyDisplayScale);
      }
      window.removeEventListener('resize', applyDisplayScale);
    };
  }, [loaded, applyDisplayScale]);

Notes:
- Deps `[loaded, applyDisplayScale]` SAFE here — this useEffect is
  declared AFTER applyDisplayScale in source order. No TDZ.
- `loaded` gate prevents firing before canvas + image are ready.
- Both visualViewport AND window listeners for defense-in-depth.
  Handler idempotent (computes desired state, not delta).
- Per pre-flight 4 stability chain: applyDisplayScale identity
  stable → deps only change when `loaded` flips false→true once
  on mount. Listener attaches ONCE.
