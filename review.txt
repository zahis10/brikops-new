THREE BUGS FIXED IN THIS BATCH
================================================================================

#1  Company name shows "לא שויך" to contractors
    Root cause: TaskDetailPage L1018 called getCompanyName(task.company_id)
    which only resolves from `projectCompanies` — that array is loaded
    inside the MGMT branch only (L240), so contractors got [] and
    fell through to "לא שויך".
    Fix:        Backend GET /tasks/{id} now returns task.company_name
                directly, resolved server-side from project_companies
                (fallback: global companies). Frontend reads it as
                first preference.

#2  Contractor sees ALL defects in their projects (cross-company leak)
    Root cause: tasks_router.py L287 strict `user['role']=='contractor'`
                gate. A contractor with user.role='user' AND
                membership.role='contractor' bypassed the filter on
                cross-project list and saw every defect.
    Fix:        Widen is_contractor detection to consider per-project
                memberships. For cross-project list, build per-project
                (project_id+company_id) $or conditions. MIXED-ROLE
                fallback so a contractor-in-A who is PM-in-B still
                sees ALL B tasks.

#3  Contractor in company can't close company-assigned defect
    Root cause: TaskDetailPage L786 + tasks_router.py L864 both
                required strict isAssignee. Defects assigned to a
                company (no specific user) gave 404 to ANY contractor.
    Fix:        Both widen to (isAssignee OR isCompanyMember).
                assignee_id is NEVER changed — WhatsApp/reminders
                still go to the original assignee only.
