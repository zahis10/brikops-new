DATA MODEL
----------
Collection: safety_project_settings (NEW)
- id, project_id (1:1), deletedAt
- developer_name, main_contractor_name, contractor_registry_number
- office_address: {city, street, house_number, postal_code,
                   email, phone, mobile, fax}
- managers: [{first_name, last_name, id_number (masked),
              id_number_hash (sha256), address, notes}]
- permit_number, form_4_target_date
- created_at, updated_at, updated_by

PII HANDLING
------------
Manager Israeli ID numbers (תעודת זהות):
- Stored as SHA-256 hash via existing `_hash_id_number` helper
  (matches workers pattern, safety_router.py:67)
- Display value masked inline as "1***789" (first 1 + last 3 chars)
- Short IDs (<4 chars) → "***" fallback
- Audit log records before/after but never raw IDs

ADAPTATIONS FROM ORIGINAL SPEC
------------------------------
Spec assumed several helpers that don't exist in our codebase. Audited
first, then adapted:
- `SAFETY_VIEWERS` constant → use `Depends(get_current_user)` directly
  (matches all existing safety GET routes; only WRITERS const exists)
- `_audit_log(db, action, actor_id=, project_id=, before=, after=)` →
  use real helper `_audit(entity_type, entity_id, action, actor_id,
  payload_dict)` with before/after wrapped in payload dict
- `_now_iso()` → `_now()` (real helper name)
- `Response` was missing from fastapi import → added

ROLE GUARDS
-----------
- GET routes: any authenticated user with project access
- PUT (upsert): SAFETY_WRITERS = (project_manager, management_team)
- Project access enforced via `_check_project_access` on every route
