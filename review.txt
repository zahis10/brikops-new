WEASYPRINT-SAFE SHIMS (no Chrome-only CSS in production output)
────────────────────────────────────────────────────────────────────
• NO display:flex with `gap` — replaced with display:table + table-cell
  for layouts: .stats (4-col), .severity-row (3-col), .info-grid (2-col),
  .meters (2-col), .sig-grid (HTML <table>), .ph header, .cover-top,
  .cover-card-strip, .tenant, .kv, .room-head, .toc-item, .sig-meta,
  .photo-row, .defect-card-head, .legal-sigline rows.
• NO CSS Grid — replaced with the same table/table-cell patterns.
• NO aspect-ratio — replaced with padding-bottom: % hack
  (.photo at 25%/3-up, .meter .photo at 56%).
• NO oklch() / no JS / no @container — none used.
• Inline SVGs have explicit width/height attrs.
• Page footer (page X / Y) implemented via @page @bottom-left/right
  running CSS counters, not in-flow elements.
• @page :first { margin: 0 } so the cover hero bleeds edge-to-edge
  without the standard inner-page margins.
• Hebrew RTL: <html dir="rtl">, body direction:rtl text-align:right;
  per-element text-align:left used only on visual-LTR slots
  (page-num counter, meta dates).

   - Tenant_2 cell rendered only when is_tenant2_required OR
     sig_t2.signed (so 1-tenant flows don't show a phantom card).

     when sig.signed is False.

     typed, "ממתין לחתימה" placeholder otherwise.
• Per-room defect cards (with photos) only render when at least one
  item has photos.
• Collapsed sections (all_not_checked / all_not_relevant): show a
  centered dashed-border banner instead of the items table.
