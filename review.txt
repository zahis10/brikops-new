PURPOSE
───────
Root-cause prevention for 2026-05-16 secrets-leak incident.
Root cause: ship.sh created commits that stayed LOCAL-ONLY; a
later `git reset --hard origin/main` orphaned them, recovery
re-wrote the files, and they were pushed publicly. 9 secrets
were rotated, ~3 hours of downtime.

FIX (3 cumulative defenses in ship.sh)
──────────────────────────────────────
1. SECRET-SCAN before commit (L49-99)
   - 7 patterns: MongoDB URI, AWS AKIA*, JWT_SECRET=value,
     Meta WhatsApp EAA*, Twilio AC*, Stripe sk_(test|live)_*,
     PEM/OpenSSH private keys.
   - Skips binaries via `file -b`.
   - Lists every offending file + reason, exits 2.
   - Override (false positives only): SHIP_SKIP_SECRET_SCAN=1

2. IMMEDIATE PUSH after commit (L105-118)
   - Pushes to `origin "$current_branch"` (NOT hardcoded main).
   - On non-FF / push failure: HALT LOUD with exit 1 + Hebrew
     instructions to rebase + re-run. No silent local-only state.

3. REMOVED hardcoded `push origin main` from sync block
   (Option B per Zahi). The old L61 would dump staging-HEAD onto
   main on every run — fixed booby trap.
   Also fixed: `git pull --rebase origin main` → `origin "$current_branch"`.
