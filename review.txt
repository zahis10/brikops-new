PART 1 — api.js : cache companies PII-TRIMMED  [option (b)]
------------------------------------------------------------------
- Added named regex const _OFFLINE_COMPANIES_RE = /\/api\/projects\/[^/]+\/companies(\?|$)/
  and pushed it into _OFFLINE_CACHEABLE (so the cache layer considers the endpoint).
- Added _trimForCache(url, data): when url matches _OFFLINE_COMPANIES_RE AND data is an array,
  maps each company to {id, name, trade} ONLY — DROPS contact_name / contact_phone /
  contact_phone_raw / contact_email. Any other url -> data returned UNCHANGED (URL-specific;
  qc/units/tasks keep full bodies).
- The ONLY cache-write site (success handler) now writes
  cachePutRead(_cacheKey(response.config), _trimForCache(response.config?.url, response.data)).
  => trimming applies ONLY to what is WRITTEN to cache; the ONLINE response (response.data)
  is untouched — the modal online still receives full company data as today.
- PII deny-list + memberships exclusion UNCHANGED. Mirrors CRIT-2.

PART 2 — NewDefectModal.js : decouple + offline-aware errors
------------------------------------------------------------------
2.1  loadCompanies: Promise.all -> Promise.allSettled. Set companies/members independently
     (normalizeList(compRes.value) / normalizeList(memRes.value)). A memberships miss can no
     longer blank companies. (allSettled never rejects -> no .catch needed; .finally kept.)
2.2  Added module-level _isOfflineFail(reason) = navigator.onLine === false || !reason?.response.
     - companies toast fires ONLY if compRes rejected AND !_isOfflineFail (a real online failure).
       A memberships-only failure -> NO toast, companies still show, assignee picker just empty.
     - Same _isOfflineFail guard added to the buildings/floors/units .catch toasts (they are
       cached so normally succeed; this silences the noise on a transient offline miss).
2.3  Empty-state copy: when companies empty AND loadError.companies (offline miss, no cache),
     the block now reads "רשימת החברות תיטען כשתחזור הרשת" + "אפשר ליצור ליקוי גם ללא שיוך קבלן"
     (no "+הוסף חברה" CTA, no implication of zero companies). The plain "אין חברות בפרויקט"
     (with the add-company CTA) is preserved for the genuine online-empty case.
2.4  Company stays OPTIONAL — no validate() / required-field change.
