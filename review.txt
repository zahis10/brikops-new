    #383 — Enrich GET /units/{unit_id}/tasks with names
    
    Backend-only change. Replaces per-task UUID slices in the
    ApartmentDashboardPage FilterDrawer with real company / assignee /
    creator names.
    
    Changes:
    - backend/contractor_ops/projects_router.py: rewrote list_unit_tasks
      - Dropped response_model=List[Task] so new fields aren't stripped
      - Added batch enrichment after the tasks query:
        * project_companies → companies fallback for company_name
        * project_memberships (per project) for assignee_name + assignee
          company_id, with users.name as fallback
        * users for created_by_name (and assignee fallback)
        * Extra project_companies → companies lookup for membership
          company_ids not already loaded from tasks
      - Total Mongo round-trips bounded at ~6 regardless of task count
      - Each response item carries company_name, assignee_name,
        assignee_company_name, created_by_name when ids resolve
    
    Architect review: initially failed because extra_company_ids only
    queried project_companies (no fallback to companies). Fixed in same
    commit by adding the companies fallback for still-missing ids.
    
    Verified via curl against demo PM account: 12-task unit returned
    populated names for company, assignee + assignee company, and
    created_by; tasks with no company/assignee correctly stay empty.
    
    Out of scope (per spec): Task pydantic schema, single-task endpoint
    (already enriched), get_unit_detail KPI endpoint, BuildingDefectsPage
    aggregate endpoint, frontend changes.
