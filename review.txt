הבאג
----
platypus.Image(...) ב‑reportlab 4.4.x דוחה ImageReader:
  TypeError: expected str, bytes or os.PathLike object, not ImageReader
_load_signature_image החזירה ImageReader, ו‑generate_tour_report מזינה אותו
ל‑flowable מסוג Image (שורה 965). הקריסה קרתה בבניית ה‑flowable / doc.build —
מחוץ ל‑try/except של ה‑helper — ולכן כל הייצוא נכשל ב‑500 וה‑FE הציג
"ייצוא PDF נכשל". משוחזר רק בסיורים עם חתימת CANVAS אחת לפחות.

התיקון (STEP 1)
---------------
_load_signature_image כעת מחזירה FILE-LIKE (io.BytesIO) במקום ImageReader:
- ענף http:  return io.BytesIO(resp.content)              (היה ImageReader(BytesIO(...)))
- ענף local: with open(str(url), "rb") as f: return io.BytesIO(f.read())
             (היה ImageReader(str(url)) — במתכונת _load_client_logo)
- ה‑try/except העוטף → None + logger.warning נשמרו בדיוק כפי שהיו (fail-soft פר‑תמונה).
- ה‑import של ImageReader הוסר מתוך ה‑helper בלבד (השימוש של _load_client_logo לא נגע).
- ה‑docstring עודכן להסביר את החזרת io.BytesIO ואת הסיבה (reportlab 4.4.x).

שורת הצרכן (965) `Image(sig_images[slot], width=3.2*cm, height=1.2*cm)` לא שונתה —
כעת מקבלת BytesIO. _load_client_logo / _load_brikops_logo (נתיב canvas.drawImage)
לא נגעו — ImageReader נכון שם.
