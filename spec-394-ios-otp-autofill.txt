# Spec #394 — iOS OTP Autofill (Associated Domains + AASA)

## בעיה

ב-iPhone ב-Capacitor WKWebView, ההצעה של iOS QuickType ל-OTP (הכפתור "מ-Messages: 123456" שמופיע מעל המקלדת) לא מופיעה.

ה-SMS מגיע עם `@app.brikops.com #{code}` כראוי, ה-input יש לו `autoComplete="one-time-code"`, אבל iOS לא מקשר בין ה-SMS לאפליקציה כי חסר Associated Domains entitlement + קובץ AASA (apple-app-site-association).

ב-Spec #393 פתרנו את אנדרואיד דרך Web OTP API. עכשיו צריך פתרון iOS-specific.

---

## קבצים לשינוי

### 1. `frontend/ios/App/App/App.entitlements`

הוסף את ה-key הבא לתוך ה-`<dict>` (אל תיגע ב-applesignin הקיים):

```xml
<key>com.apple.developer.associated-domains</key>
<array>
    <string>webcredentials:app.brikops.com</string>
</array>
```

הקובץ הסופי צריך להיראות **בדיוק** כך:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
	<key>com.apple.developer.applesignin</key>
	<array>
		<string>Default</string>
	</array>
	<key>com.apple.developer.associated-domains</key>
	<array>
		<string>webcredentials:app.brikops.com</string>
	</array>
</dict>
</plist>
```

### 2. `frontend/public/.well-known/apple-app-site-association` (קובץ חדש, **בלי סיומת**)

תוכן (JSON תקני, ללא רווחים מיותרים בהתחלה/סוף):

```json
{
  "webcredentials": {
    "apps": ["8FV5CZ886X.com.brikops.app"]
  }
}
```

**חשוב:**
- שם הקובץ הוא **בדיוק** `apple-app-site-association` — בלי `.json`, בלי `.txt`, בלי שום סיומת.
- התיקייה היא `.well-known` (עם נקודה בהתחלה).
- Team ID `8FV5CZ886X` מאומת מתוך `frontend/ios/App/App.xcodeproj/project.pbxproj` (מופיע כ-`DEVELOPMENT_TEAM`).
- bundle ID `com.brikops.app` מאומת מתוך `frontend/capacitor.config.json` ו-`frontend/ios/App/App.xcodeproj/project.pbxproj`.

## אחרי שדחפת (Replit's responsibility)

- ספור עד 30 שניות — Cloudflare Pages צריך לתפוס את ה-deploy של ה-frontend
- בדוק שה-URL של AASA זמין:
  ```bash
  curl -I https://app.brikops.com/.well-known/apple-app-site-association
  ```
  ציפיות:
  - `HTTP/2 200`
  - `Content-Type: application/json` (אם זה `text/html` או `text/plain` — יש בעיה ב-Cloudflare)
  - ללא redirect
- אם ה-Content-Type לא JSON — זה issue של Cloudflare Pages routing, נצטרך להוסיף `_headers` rule:
  ```
  /.well-known/apple-app-site-association
    Content-Type: application/json
  ```
  בקובץ `frontend/public/_headers`.

## חשוב: זה NATIVE CHANGE

App.entitlements הוא תחת `frontend/ios/` — לפי `CLAUDE.md` של הפרויקט, **כל שינוי תחת `ios/` דורש ship.sh + build חדש**, לא Capgo OTA. אז:

- Replit: רק dom/push (deploy.sh ל-Cloudflare ידחוף את ה-AASA)
- Zahi (במק): `./ship.sh` → Xcode → Archive → Upload to TestFlight → להתקין על האייפון
- בלי build חדש על האייפון — entitlement לא יחול והתיקון לא יעבוד
